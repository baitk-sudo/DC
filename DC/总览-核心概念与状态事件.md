# 总览（一页骨架）— 核心概念、状态与触发

> 目的：内建立全局心智模型

---

## 1. 一句话定位

本系统以**进件（Intake）**为流程载体，用“**状态机 + 配置驱动（产品/规则/同步策略/路由/签约轮次）**”支撑多机构多产品；风控贯穿贷前到贷后；批处理提供批量生成、扫描与补偿重放。

---

## 2. 核心业务对象（不讲术语版）

| 对象 | 你们文档中的主要归属 | 作用 |
|---|---|---|
| 进件 | `进件管理组件` + `进件中心-业务数据同步策略设计` | 申请与流程载体、状态机、路由、同步策略触发 |
| 客户 | 第2段 | 临时/正式、对私/对公、多源建档与标签 |
| 授信 / 用信 / 额度 | 第5段 + 第7段 | 授信结论、用信记录、额度台账/调整/激活/占用 |
| 合同 / 协议 / 签约任务 | 第6段 | 授信合同/借款合同/放款申请书/授权协议等，多轮签约与两类入口（面客/办贷PAD） |
| 放款指令 / 受托支付审批 | 第7段 | 提款、校验、受托支付审批、核心放款、回盘与对账 |
| 还款计划 / 还款指令 | 第8段 | 计划生成、计息/罚息、入账/冲账、逾期维护 |
| 预警工单 / 风险缓释动作 | 第9段 | 预警规则、工单闭环、额度冻结/下调、评级下调、提前到期等 |
| 报表与报送批次 | 第10段 | 统一数据出口：监管报送 + 经营分析 |

---

## 3. 进件状态机（全流程骨架）

> 以你们《进件中心-业务数据同步策略设计.md》为准，本表只给“看懂全局”的最小解释。

| 状态 | 含义 | 典型触发方 | 典型动作（按产品配置） |
|---|---|---|---|
| `RECEIVED` | 已接收 | 进件管理 | 早同步产品：初始化客户/评级/授信/用信/额度等；触发进件准入风控与路由 |
| `UNDER_REVIEW` | 审批/调查中 | 工作流/审批流 | 转人工三合一、审批流流转、补录与尽调 |
| `APPROVED` | 审批通过 | 工作流/审批流 | 生成合同草稿/待签、生成签约任务（或进入直通签约池） |
| `SIGNED` | 合同签约完成 | 签约组件 → 进件管理 | **同步策略核心触发点**：晚同步一次性落地；早同步做更新/终态确认 |
| `DISBURSED` | 已放款 | 放款组件 → 进件管理 | 放款结果回写、必要的终态确认与对账（按策略） |
| `REJECTED` | 已拒绝 | 风控/审批流 | 终止流程，留痕与回溯 |

---

## 4. 关键触发点（事件/状态变化 → 影响什么）

> 不要求实现“事件总线”，这里的“触发点”用于统一对齐：谁在什么节点驱动了哪些后续动作；失败如何补偿（与第11段批处理衔接）。

| 触发点 | 触发组件 | 直接产物 | 主要影响 |
|---|---|---|---|
| 进件接收完成（进入 `RECEIVED`） | 进件管理 | 进件单落库 | 触发（可选）早同步；触发进件准入风控；进入路由分支 |
| 进件准入风控完成（R1/R3/拒绝） | 风控组件 | 准入结论 + 标签 | R1 直通生成签约任务；R3 转人工生成三合一/审批任务；拒绝则终止 |
| 三合一提交 / 审批完成 | 授信管理 + 工作流 | 审批结论 | 推动进件进入 `APPROVED`，生成合同与待签任务 |
| 合同/协议签署完成 | 签约组件 + 合同管理 | 合同生效/归档、签约任务已签 | 通知进件管理进入 `SIGNED` 并执行同步策略（最关键） |
| 提款/放款成功回盘 | 放款组件 | 放款成功流水 | 用信状态更新、额度占用更新、进件进入 `DISBURSED` |
| 还款入账/冲账完成 | 还款组件 | 还款流水、计划更新 | 用信状态（结清/逾期）更新；贷后预警输入 |
| 贷后预警命中 | 风控（贷后规则包）/贷后组件 | 预警等级、工单 | 触发工单与处置，必要时风险缓释（额度/授信/评级） |
| 报表生成/报送 | 报表组件 + 批处理 | 报表版本/批次 | 监管/经营输出；可选审核流 |

## 5. 三张“配置驱动”总表

> 你们的《产品流程-组件调用场景视图与配置表.md》里已经把配置拆成多张表。汇报时建议只先讲清三类配置，细节再落到原表。

- **产品主配置**：客户类型、是否允许未注册申请、是否早同步、R1/R3 路由、多轮签约、受托支付审批、额度激活/调整等
- **同步策略配置**：在 `RECEIVED/APPROVED/SIGNED/DISBURSED` 哪些状态对哪些模块做 初始化/更新/终态确认（按进件幂等 upsert）
- **授权/风控/路由配置**：阶段 × 规则包 × 授权检查模式 × 路由条件（额度阈值、风险结果等）
