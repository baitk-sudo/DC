# 组件设计分析 — 组件调用关系与前置条件总览

本文件从**组件视角**总结每个组件的：**前置调用条件**、**前置调用者**、**后续调用对象**，方便在实现阶段一眼看出组件的责任边界与依赖前提。业务细节、字段含义与多产品支持点仍以各段落设计文档为准。

> 说明：前置调用条件聚焦于「组件被调用前必须满足的状态/上下文」，不展开所有业务规则。

---

## 一、渠道端组件

| 组件名称 | 前置调用者（示例） | 前置调用条件（被调用前至少满足） | 后续调用组件（示例） |
|----------|--------------------|-----------------------------------|----------------------|
| 用户注册组件 | 渠道前端（APP/H5/微信/合作平台）、外部平台推送 | 已选定注册渠道与入口（或外部平台标识）；具备最小身份标识（手机号/证件号等）；系统管理中已配置对应注册/认证策略；如需人脸/短信等，多因子策略已生效 | 客户管理组件（创建/更新临时客户）、系统管理组件（认证策略）、消息中心/人脸识别/安全组件（验证码、人脸、密码） |
| 客户授权准入组件 | 渠道前端（独立授权入口）、授信/贷款申请组件 | 已选定产品、客户类型、业务阶段（AUTHORIZATION/LOAN_APPLY/CREDIT_APPLY 等）；客户及/或关联人已登录/可识别；授权配置管理中针对「产品+客户类型+阶段」已存在配置（授权文件+适用关联人+规则包 + 授权检查模式）；若为补授权场景，可查询到历史授权记录用于提示，并根据“授权检查模式”（强制/仅提示/仅记录）决定缺失授权时是直接拦截还是仅提示后允许继续 | 授权配置管理组件（查授权文件+规则包 ID 列表及检查模式）、签约组件（授权协议签署）、风控组件（执行授权准入规则包）、征信与外部数据管理组件（在授权之后拉取数据）、进件管理组件（写授权结果到进件/申请） |
| 预授信组件 | 渠道前端、营销方案管理组件、批处理组件、征信与外部数据管理组件（名单推送） | 已确定预授信产品及客户（或名单）；若按客户维度预授信，客户需具备基本身份信息；系统已配置对应预授信规则包与额度策略 | 风控组件（预授信规则执行/白名单校验）、征信与外部数据管理组件（外部数据查询）、额度管理组件（写预授信额度）、营销方案管理组件（展示预授信结果） |
| 授信申请组件 | 渠道前端（产品详情页） | 调用时须能唯一识别客户：要么已有已注册/已认证的客户标识（如 customerId），要么本次申请过程中采集到足够的身份信息，可由客户管理组件创建临时客户；**若客户类型为对公，则在发起授信申请前必须已完成对公客户注册与基础认证（由用户注册组件 + 客户管理组件完成）**；已选定授信产品，且产品管理中已配置授信申请要素与校验规则；如产品要求“先授权后申请”，需能通过授权准入组件或授权配置验证该前置条件 | 客户授权准入组件（如本阶段需授权）、进件管理组件（生成授信进件）、客户管理组件（创建/补录或更新客户信息，含临时客户）、系统管理/消息中心/人脸识别/安全组件（认证与风控前置校验） |
| 贷款申请组件 | 渠道前端 | 调用时须能唯一识别客户：要么已有登录态和客户标识，要么在申请过程中可通过用户注册组件 + 客户管理组件创建临时客户；**若客户类型为对公，则在发起贷款/用信申请前必须已完成对公客户注册与基础认证（由用户注册组件 + 客户管理组件完成）**；已选定贷款/用信产品及目标授信/额度项（如适用）；产品管理中已配置贷款申请要素与规则；如产品要求“先预授信/先授权”，需能通过预授信组件或授权准入组件验证对应前置条件，否则可按“直接贷款”模式进入进件 | 客户授权准入组件（如本阶段需授权）、进件管理组件（生成用信进件）、用信管理组件（初始化用信申请）、额度管理组件（如有授信/额度时校验可用额度）、消息中心/人脸识别/安全组件 |
| 提额组件 | 渠道前端、办贷端入口 | 已能唯一定位提额对象：授信提额需存在授信额度记录，贷款提额需存在目标贷款/用信记录；**若提额对象为对公客户或对公授信/用信，则在发起提额前必须已完成对公客户注册与基础认证（由用户注册组件 + 客户管理组件完成）**；产品/额度管理中已开放提额入口并配置提额规则；若需先授权或先完成特定贷后检查，则相应条件已满足或在流程中通过授权准入/贷后管理验证 | 进件管理组件（生成提额进件）、风控组件（提额风控规则）、额度管理组件（调整额度）、工作流/审批流组件（高风险/高额度提额审批） |
| 股东会决议组件 | 渠道前端（企业负责人入口）、办贷端 | 已识别对公客户与企业负责人；已配置股东会决议适用的产品与场景；已具备股东名单和通信方式（手机号、邮箱等） | 签约组件（生成并下发股东会决议签约任务）、合同（协议）模版管理组件（决议模板）、合同管理组件（决议归档）、征信与外部数据管理组件（工商/司法数据交叉验证）、消息中心组件（短信/邮件通知股东） |
| 签约组件 | 渠道前端、办贷端 PAD、进件管理组件、工作流/审批流组件、授信管理组件、放款组件 | 已存在有效的签约任务或可构造待签任务的上下文（合同实例 ID 或生成条件、进件 ID、客户/关联人信息、产品与任务类型）；如产品要求签约前风控/授权，需可获取对应规则包 ID 或在签约流程中触发 | 合同管理组件（获取/更新合同实例）、合同（协议）模版管理组件（模板结构）、风控组件（签约前风控，可选）、消息中心/人脸识别/安全组件（验证）、进件管理组件（签约完成事件驱动 SIGNED 与同步策略） |
| 放款组件（渠道端） | 渠道前端 | 客户已签署必要合同/放款申请书（视产品而定）；用信管理组件中存在可提款的用信/合同记录；额度管理组件中有可用额度；系统管理中已配置提款前安全验证策略（短信/人脸/密码等） | 放款组件（办贷端）（传递放款指令）、用信管理组件（可提款额度与用信状态查询）、额度管理组件（可用额度校验）、合同管理组件（放款申请书）、签约组件（如需签放款申请书）、消息中心/人脸识别/安全组件（提款验证） |
| 还款组件（渠道端） | 渠道前端 | 客户已登录且在用信管理中存在贷款/用信记录；还款计划已生成或可按需计算；系统管理中已配置还款授权与扣款规则（支付渠道、关联账户等） | 还款组件（办贷端）（发起还款指令）、用信管理组件（还款计划与用信状态查询）、利率与费用组件（应还额展示用可选）、消息中心组件（结果通知） |

## 二、办贷端域组件

| 组件名称 | 前置调用者（示例） | 前置调用条件 | 后续调用组件（示例） |
|----------|--------------------|--------------|----------------------|
| 系统管理组件 | 运维/管理员、其他所有组件 | 系统基础安装完成；需要有初始管理员账户 | 消息中心、人脸识别、安全组件（策略）、各业务组件（权限与参数查询） |
| 产品管理组件 | 运维/产品配置人员、进件管理组件、授信/用信/合同管理组件 | 已定义机构、币种、客户类型等基础枚举；产品上线前必须完成要素与参数配置 | 进件管理组件、授信管理组件、用信管理组件、利率与费用组件、营销方案管理组件等（查询产品参数） |
| 营销方案管理组件 | 运维/市场人员、渠道前端 | 产品管理中已有可售产品；客户标签/白名单数据可用 | 渠道前端（产品列表/详情展示）、预授信组件（批量预授信触发）、征信与外部数据管理组件（客群筛选）、批处理组件（预授信与营销任务调度） |
| 授权配置管理组件 | 客户授权准入组件、进件管理组件、签约组件 | 产品、客户类型、业务阶段参数已传入；后台已维护对应「产品+客户类型+阶段」下的授权文件配置（含适用关联人类型、是否必签）与风控规则包绑定关系；合同（协议）模版管理组件中已存在对应模板编码 | 合同（协议）模版管理组件（通过授权文件编码取模板）、风控组件（消费规则包 ID 列表） |
| 进件管理组件 | 渠道端申请组件、工作流/审批流组件、签约组件、放款组件 | 传入的申请/进件数据完整（至少包含产品、进件类型、客户标识、基本业务字段）；对应产品在同步策略表中已有配置；进件状态机定义完整；如产品要求“先授权/先预授信”，需在进件创建或进入关键状态前可通过授权/预授信组件或配置验证该前置条件 | 风控组件（进件风控准入）、工作流/审批流组件（路由与任务生成）、客户/评级/授信/用信/额度/合同管理组件（按状态与策略调用 upsertByIntake）、监管报送与报表组件（通过数据视图间接消费） |
| 客户管理组件 | 用户注册组件、进件管理组件、办贷端录入、征信与外部数据管理组件 | 至少具备基础身份/组织信息；需能区分个人/对公及临时/正式客户状态；外部来源需提供 external_source + external_customer_id | 评级管理、授信管理、用信管理、额度管理、合同管理、贷后管理、监管报送与报表组件（均以客户号为关联键） |
| 评级管理组件 | 进件管理组件、三合一任务、贷后管理组件、风控组件（作为规则输入） | 客户信息与必要的财务/行为数据已准备好（可来自征信、外部数据、内部行为）；已配置评级模型与生效策略 | 授信管理组件（授信额度与条件参考）、额度管理组件（额度策略参考）、贷后管理组件（评级调整）、监管报送与报表组件（评级结果统计） |
| 授信管理组件 | 三合一任务、进件管理组件、贷后管理组件 | 已有进件或授信申请记录；必要的评级、押品、风控结果可用（或在三合一节点内获取）；产品配置中定义了授信额度、期限、担保等规则 | 合同管理组件（生成授信合同）、签约组件（授信合同签约任务）、额度管理组件（授信额度落库）、进件管理组件（状态更新）、贷后管理/监管报表组件（授信数据消费） |
| 额度管理组件 | 授信管理组件、进件管理组件、放款组件、贷后管理组件 | 已有授信或预授信基础；产品配置定义了额度维度（客户/授信/产品/机构等）与占用策略；若执行调整/激活，通常需工作流审批通过 | 放款组件（可用额度校验）、进件管理/工作流（额度阈值路由）、贷后管理组件（风险缓释动作）、监管报送与报表组件（集中度报表） |
| 用信管理组件 | 贷款申请组件、进件管理组件、放款组件、还款组件、贷后管理组件 | 已有授信或可用额度；用信申请数据完整（金额、期限、用途、担保等）；合同管理已或将生成对应合同 | 放款组件（放款状态与可提款额度）、还款组件（还款计划生成与执行状态）、额度管理组件（额度占用/释放）、贷后管理组件（逾期/五级分类输入）、监管报送与报表组件 |
| 合同管理组件 | 进件管理组件、授信管理组件、放款组件、签约组件 | 已有产品、客户、授信/用信/进件等业务主键；合同模板已在合同（协议）模版管理组件中配置；合同生成/生效策略已定义 | 签约组件（合同内容展示与签署结果回写）、进件管理组件（在 SIGNED/其它状态下做终态确认）、监管报送与报表组件（合同视图）、贷后管理组件（合同维度信息） |
| 合同（协议）模版管理组件 | 授权配置管理组件、合同管理组件、签约组件 | 模板元数据已配置（合同/协议类型、编码、版本、适用产品/机构/客户类型）；存储/渲染引擎可用 | 合同管理组件（实例生成）、签约组件（展示模板结构）、授权配置管理组件（通过编码引用模板） |

| 放款组件（办贷端） | 放款组件（渠道端）、工作流/审批流组件（受托支付审批）、内部系统 | 合同已生效；用信记录存在且允许放款；额度管理中有足够可用额度；若产品配置“放款前风控”或“受托支付审批”，需取得对应风控结果/审批通过状态或在流程中触发；核心/账务接口配置就绪 | 用信管理组件（更新放款状态）、额度管理组件（额度占用更新）、合同管理组件（合同状态确认）、风控组件（放款前风控）、工作流/审批流组件（受托支付审批）、核心/账务系统（放款交易）、进件管理组件（状态 DISBURSED） |
| 还款组件（办贷端） | 还款组件（渠道端）、内部系统、批处理组件 | 用信记录存在且处于需还款状态；还款计划存在或可生成；核心/账务接口配置就绪；若涉及减免/冲正，通常需审批结果 | 用信管理组件（更新还款执行状态与结清标志）、利率与费用组件（应还本息费/罚息计算）、核心/账务系统（入账）、贷后管理组件（逾期与处置数据）、监管报送与报表组件 |
| 风控组件 | 预授信组件、客户授权准入组件、进件管理组件、三合一任务、签约组件、放款组件、贷后管理/预警组件、批处理组件 | 对应阶段的规则包 ID 列表已通过授权配置管理/产品配置等确定；必要输入数据（客户/进件/授信/用信/押品/外部数据等）可获取或通过征信与外部数据管理组件拉取；外部数据源与风控引擎自身处于可用状态 | 征信与外部数据管理组件（取数）、额度管理组件（部分额度测算场景）、调用方业务组件（消费风控结论：R1/R3/拒绝/预警等级等） |
| 利率与费用组件 | 授信管理组件、用信管理组件、合同管理组件、还款组件、批处理组件 | 对应产品在产品管理中已配置利率与费用规则（基准利率、浮动、费用项、罚息规则等）；如有优惠/调整规则，对应配置已生效；调用方提供了明确的计算上下文（授信/用信/合同主键及时间点、金额等） | 授信管理组件（授信定价结果）、用信管理组件/合同管理组件（用信/合同利率与费用）、还款组件（应还本息费/罚息计算）、监管报送与报表组件（收益等统计） |
