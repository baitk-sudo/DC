# 如何将 MD 文档发布到服务器

本目录已配置为 **Docsify** 文档站点，所有 `.md` 文件无需编译，可直接在浏览器中渲染。按以下方式之一即可发布到服务器。

---

## 一、通过 Git 发布并分享（推荐：他人用浏览器即可访问）

把文档推到 **GitHub**（或 Gitee / GitLab）并开启 **Pages**，即可得到一个可分享的网址，任何人用浏览器打开即可查看。

### 1. 确认要发布的内容在 Git 里

当前文档和站点文件在目录 `DC` 下。若你的 Git 仓库**根目录就是 DC**（即仓库里直接是 `index.html`、`_sidebar.md`、各 `.md` 等），则无需改目录；若仓库根目录是上级（例如 AICoding），而 DC 只是其中一个子目录，则需要让 Pages 从 **DC 所在目录** 发布（见下）。

在仓库根目录执行：

```bash
git add DC/index.html DC/_sidebar.md DC/发布说明.md
git add DC/*.md
git status
git commit -m "docs: 增加 Docsify 站点与发布说明"
git push origin main
```

（如仓库根就是 DC，把上面路径里的 `DC/` 去掉即可。）

### 2. 在 GitHub 上开启 Pages

1. 打开仓库页面：**https://github.com/baitk-sudo/DC**
2. 点击 **Settings（设置）** → 左侧 **Pages**
3. 在 **Build and deployment** 里：
   - **Source** 选 **Deploy from a branch**
   - **Branch** 选 `main`（或你实际使用的分支）
   - **Folder** 选 **/ (root)**（若仓库根就是 DC 且已有 `index.html`）；若文档在子目录 `docs`，则选 **/docs**
4. 点 **Save**，等 1～2 分钟

### 3. 访问与分享

发布成功后，站点地址一般为：

- **https://baitk-sudo.github.io/DC/**

把该链接发给同事，对方用浏览器打开即可查看全部文档。之后你只需 `git push` 更新内容，Pages 会自动更新页面。

### 若使用 Gitee 或 GitLab

- **Gitee**：仓库 → **服务** → **Gitee Pages** → 选择分支与目录 → 开启，获得 `https://你的用户名.gitee.io/DC/` 之类地址。
- **GitLab**：仓库 → **Deploy** → **Pages**，按提示从分支发布，获得 `https://你的用户名.gitlab.io/DC/` 之类地址。

## 二、本地预览（发布前可先看效果）

在项目根目录（即本目录）执行：

```bash
npx docsify-cli serve .
```

默认访问：<http://localhost:3000>。用于确认侧栏与链接是否正常。

---

## 三、发布到静态服务器

将**整个 DC 目录**（含 `index.html`、`_sidebar.md` 及所有 `.md` 文件）上传到任意支持静态托管的服务器即可。

### 方式 A：Nginx / Apache 等

- 把本目录内容拷贝到站点根目录（如 `/var/www/docs` 或 `C:\inetpub\wwwroot\docs`）。
- 确保站点默认首页为 `index.html`，且对 `.md` 请求由 Docsify 前端路由处理（即访问根或任意路径都返回 `index.html`）。
- **Nginx 示例**（站点根目录为 `/var/www/dc-docs`）：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /var/www/dc-docs;
    index index.html;
    location / {
        try_files $uri $uri/ $uri.html /index.html;
    }
}
```

### 方式 B：内网文件服务器

- 若服务器仅提供静态文件（如通过 SMB/FTP 上传），可将 DC 目录放到 Web 服务器配置的静态目录下，并保证通过 HTTP 能访问到 `index.html`。

### 方式 C：GitHub Pages / GitLab Pages

1. 将本仓库推送到 GitHub 或 GitLab。
2. 在仓库设置中开启 Pages，发布分支选择 `main`（或当前分支），**发布源选择“根目录”或本 DC 所在目录**。
3. 若使用 GitHub Pages 且站点在子目录（如 `docs`），在仓库设置里将 Pages 的 “Source” 设为该目录；根目录下保留 `index.html` 和 `_sidebar.md` 即可。

---

## 四、注意事项

- **编码**：所有 `.md` 请使用 **UTF-8** 保存，避免中文乱码。
- **链接**：侧栏与正文中的链接均为相对路径，部署时保持当前目录结构不变即可。
- **CDN**：`index.html` 通过 CDN 加载 Docsify 与主题，服务器需能访问外网（或自行替换为内网/本地静态资源）。
- **仅内网时**：若服务器不能访问外网，可将 [Docsify 官方构建文件](https://github.com/docsifyjs/docsify/releases) 下载到本目录（如 `lib/`），并修改 `index.html` 中的 `href`/`src` 为本地路径。

## 五、目录与文件说明

| 文件/目录     | 说明 |
|--------------|------|
| `index.html` | Docsify 入口页，由浏览器直接打开或由 Web 服务器提供。 |
| `_sidebar.md`| 侧栏目录，决定站点左侧导航结构。 |
| `*.md`       | 正文文档，无需编译，按需增删或修改。 |
| `发布说明.md`| 本说明文件。 |

更新文档时只需修改或新增 `.md` 文件；若需调整导航，编辑 `_sidebar.md` 即可，无需重新“构建”。
