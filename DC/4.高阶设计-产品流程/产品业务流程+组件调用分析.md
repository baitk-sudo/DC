# 产品业务流程 + 组件调用分析（表样与取数规则）

> 用于把「产品业务流程」与「组件调用/前置条件」关联起来，形成可按产品逐行填充的分析表。  
> **权威数据源**：`1.产品经理输入：产品业务流程/产品业务流程与组件综合分析.csv` 为**唯一准表**；本分析表的业务阶段、进入组件、进入组件内部处理逻辑、**说明**列均以该 CSV 为准，并反向驱动 `产品流程-组件调用场景视图与配置表.md` 等设计文档中的「说明」列与阶段划分。

---

## 一、数据来源与字段含义

本分析表的字段与取数规则以 **`产品业务流程与组件综合分析.csv`** 为准，各列表头含义如下：

| 列名 | 含义 | 数据来源 |
|------|------|----------|
| 营销产品 | 产品名称（营销/业务视角） | **CSV「营销产品」列** |
| 业务阶段 | 产品办理过程中的业务阶段/步骤 | **CSV「业务阶段」列**（含可选阶段如提额、白名单维护等） |
| 前置组件入口 | 在该业务阶段发起调用的入口组件（渠道或办贷） | **CSV「前置组件入口」列** |
| 前置调用条件（被调用前至少满足） | 触发入口组件前必须满足的业务/技术前提 | **CSV「前置调用条件」列**；可与 `组件设计-组件调用关系总览.md` 对齐，详细条目以总览为准 |
| 进入组件 | 在该阶段被真正「进入」或需要分析的组件名称 | **CSV「进入组件」列** |
| 进入组件内部处理逻辑 | 被进入组件在该阶段内部如何调用其他组件、进行处理 | **CSV「进入组件内部处理逻辑」列** |
| 说明 | 针对该阶段与该组件调用的额外说明、产品特性或差异点；**晚同步/早同步、R1/R3、黑名单拦截时机、申请内授权/独立授权、提额、白名单维护等均在此列体现** | **CSV「说明」列**；设计文档（视图、回归矩阵、步骤级分析）中的说明须与此列一致，不脱节 |

> **说明列驱动**：CSV 的「说明」列若写了晚同步/早同步、R1/R3、黑名单在何阶段拦截、申请内授权/独立授权、提额可选、白名单维护等，须在 `产品流程-组件调用场景视图与配置表.md`、产品规则回归矩阵、步骤级分析等文档中一致体现。
