# 数字化信贷系统 — 组件设计与产品流程分析

本项目为**数字化信贷系统**的组件架构与业务段落设计文档，按业务办理过程顺序拆分为多段组件设计，并配套进件同步策略、产品流程步骤级分析及组件清单。

## 快速入口（推荐）

- **汇报版导航（先见森林再见树木）**：打开 `index-report.html#/`
- **实施版导航（开发查阅更顺）**：打开 `index-impl.html#/`
- **一页骨架（5～10 分钟看懂全局）**：汇报版首页默认打开 [overview.md](overview.md)（根目录，与横切设计中「总览-核心概念与状态事件」内容一致，便于稳定访问）

> 说明：本仓库提供两套侧栏（`sidebar-report.md` / `sidebar-impl.md`），用于“汇报动线”和“实施查阅”两种阅读习惯；文档内容本身不重复。

| 视角 | 适用场景 | 入口 |

## 设计原则

- **配置驱动**：支持多产品、多机构，通过配置减少二次开发。
- **进件驱动同步**：进件状态（RECEIVED / APPROVED / SIGNED / DISBURSED）驱动各业务模块的写入与更新，支持早同步与晚同步两种模式。
- **签约为公共能力**：签约组件统一支撑授权协议与合同签订，支持面客渠道与办贷端 PAD 现场签约。
