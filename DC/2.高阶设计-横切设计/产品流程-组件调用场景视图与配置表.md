# 产品流程 × 组件调用场景视图与配置表

本文档将各产品的业务办理流程与《组件设计-组件调用关系总览》对齐，形成：**按产品的组件调用场景视图** 与 **实施时需维护的配置表**，便于按产品落地实现与配置管理。

---

## 一、组件调用场景视图（按产品）

以下按产品列出「业务阶段 → 主调组件 → 被调组件链」的调用关系，与总览中的前置条件一致；表中「主调」为流程入口或编排方，「被调链」为在该阶段内依次或并行调用的组件。

### 1.1 湖北农信-公积金贷款（线上）

| 业务阶段 | 主调/入口 | 组件调用链（主调→被调） | 说明 |
|----------|-----------|--------------------------|------|
| 1、客户注册 | 渠道前端 | 用户注册组件 → 客户管理组件（临时客户）、系统管理/消息/人脸/安全 | 先注册认证再申请；个人客户 |
| 2、查询产品信息 | 渠道前端 | 营销方案管理组件 → 产品管理组件、利率与费用组件、额度管理组件、合同管理组件、预授信组件 | 展示产品列表/详情、利率优惠券、额度/合同可用额，并通过预授信组件展示预计可贷款金额，通过产品管理组件展示产品最高可贷款金额；本阶段不做黑/白名单拦截，是否可发起申请由后续申请前规则决定 |
| 3、客户发起贷款申请 | 渠道前端 | 贷款申请组件 → 风控组件（**申请前规则包：反欺诈、黑/白名单、产品参数准入**，按产品配置）→ 客户管理组件（补录）→ 客户授权准入组件 → 授权配置管理组件 → 合同管理组件（授权协议可展示内容）→ 消息/人脸 → 签约组件（授权）→ 进件管理组件 | 提交前按产品配置执行申请前规则包：未通过则在本阶段直接拦截（不创建进件、也不进入授权）；通过后才进入授权准入与后续进件；授权协议展示由合同管理提供 |
| 4、进件接收 | 进件管理组件 | 进件管理组件 → **授权配置管理组件（查询进件阶段规则包 ID）** → 客户管理组件（补录）→ 风控组件（准入，规则包由授权配置提供）→ [R1]签约组件（生成签约任务）/ [R3]进件管理组件（转线下，按配置办贷任务类型） | 晚同步：SIGNED 前不写客户/评级/授信/用信；进件准入规则包从授权配置（产品+阶段）取 |
| 5、客户在线签借款合同 | 渠道/进件 | 签约组件 → 合同管理组件（可展示内容）→ 消息/人脸/安全 → 合同管理组件（签署回写）→ 客户/用信/授信/评级/合同管理组件（SIGNED 时同步）→ 进件管理组件（状态） | 签约完成触发晚同步一次性写业务表；展示内容由合同管理提供 |
| 6、客户发起提款 | 渠道前端 | 放款组件（渠道）→ 用信管理/额度管理/合同管理组件 → 安全/人脸/消息 → 签约组件（放款申请书）→ 放款组件（办贷）→ 核心系统 → 进件管理组件 | 提款前风控/验证按产品配置 |
| 7、受托支付审批 | 办贷/工作流 | 工作流/审批流组件 → 放款组件（办贷） | 按产品配置是否启用 |
| 8、客户还款 | 渠道前端 | 还款组件（渠道）→ 用信管理组件/利率与费用组件 → 还款组件（办贷）→ 核心/贷后/监管 | — |

### 1.3 湖南农信-公积金贷款

| 业务阶段 | 主调/入口 | 组件调用链（主调→被调） | 说明 |
|----------|-----------|--------------------------|------|

| 4、系统进件接收 | 进件管理组件 | 进件管理组件 → **授权配置管理组件（查询进件阶段规则包 ID）** → 客户管理组件（创建）→ 风控组件（准入，规则包由授权配置提供）→ 评级管理组件 → 风控组件（额度）→ 授信管理组件（初始化）→ 用信管理组件（初始化）→ [≤20W]合同管理组件+签约组件+进件管理组件 / [>20W]进件管理组件（转线下） | **早同步**：RECEIVED 即初始化客户/评级/授信/用信；**进件准入规则包 ID 从授权配置（产品+阶段）取** |

### 1.4 长沙银行-税E贷

| 业务阶段 | 主调/入口 | 组件调用链（主调→被调） | 说明 |
|----------|-----------|--------------------------|------|
| 1、预授信申请 | 办贷/外部 | 预授信组件 → 风控组件 ← 征信与外部数据管理组件 | 白名单维护或推送 |
| 2、客户注册认证 | 渠道前端 | 用户注册组件 → 客户管理组件、系统管理/消息/人脸/安全 | **对公客户办理业务需要先在渠道端先完成注册认证**，未完成不得进入后续授权/授信/进件流程 |
| 3、客户查询产品信息 | 渠道前端 | 营销方案管理组件 → 产品管理组件、利率与费用组件、额度管理组件、合同管理组件、预授信组件、名单管理组件 → 客户授权准入组件 | 在产品列表/详情、优惠券、额度/合同可用额查询的基础上，增加两类额度展示服务：1）调用**预授信组件**查询预授信额度（预计可贷款金额）；2）调用**产品管理组件**查询产品最高可贷款额度；同时增加**前置检查：判断产品是否有黑/白名单限制及申请客户是否在名单中**；检查通过后从产品详情页进入独立的“客户授权准入”阶段 |
| 4、客户授权准入 | 渠道前端 | 客户授权准入组件 → 授权配置管理组件 → 合同管理组件（授权协议展示）/消息 → 签约组件（授权）→ 风控组件（授权准入规则） | **独立授权阶段**，先授权后授信申请 |
| 5、客户发起授信申请 | 渠道前端 | 授信申请组件 → 进件管理组件（授信进件） | 授信与贷款进件分离 |
| 6、股东会决议 | 企业负责人/办贷 | 股东会决议组件 → 征信与外部数据管理组件（工商/司法）→ 签约组件 → 合同管理组件（含模板与归档）、消息中心 | 对公股东会决议签署；先查股东成员再下发签署任务；展示内容由合同管理提供 |
| 6、系统进件接收 | 进件管理组件 | 进件管理组件 → 客户管理组件 → 风控组件 → 评级管理组件 → 风控组件（额度）→ 授信管理组件（初始化）→ 用信管理组件（初始化）→ 进件管理组件（分配） | **早同步**；授信进件 |
| 7、客户经理补录客户信息 | 办贷端 | 客户管理组件 | 建档/尽调补录 |
| 8～10、利率/额度调整与额度激活 | 办贷端 | 授信管理/额度管理组件 → 工作流/审批流组件 → 利率与费用组件/额度管理组件 | 显式额度激活与用信分离 |
| 11～12、三合一与审批 | 办贷端 | 授信管理组件 → 评级管理/授信管理/用信管理/风控组件 → 工作流/审批流组件 → 合同管理组件 → 签约组件 | 生成授信合同与签约任务 |
| 13、客户在线签授信合同 | 渠道/进件 | 签约组件 → 合同管理组件（可展示内容）/消息/人脸 → 用信管理/授信管理/评级管理/合同管理组件（更新）→ 进件管理组件 | 多轮签约：先授信合同；展示内容由合同管理提供 |
| 14、客户发起提款 | 渠道前端 | 放款组件（渠道）→ 用信管理/额度管理/合同管理组件 → 安全/人脸/消息 → 签约组件（放款申请书）→ 放款组件（办贷）→ 核心 → 进件管理组件 | 提款即用信 |
| 15、受托支付审批 / 16、还款 | 同 1.1 | 工作流/审批流组件 → 放款组件；还款组件链 | 按配置 |

## 三、使用说明

- **组件调用场景视图**：按产品、按业务阶段查看「谁在什么环节调用谁」，与《组件设计-组件调用关系总览》中的前置条件一致；实现时需保证调用顺序与前置条件满足关系一致。
- **配置表**：实施时在系统中维护对应配置（或通过参数表/配置中心落地）；新增产品时在 2.1～2.5 各表中增加一行或若干行，并同步更新《产品规则-回归用例矩阵》与本文档的视图。
- **对公统一前置**：对公客户办理业务需要先在渠道端先完成注册认证；客户类型为对公（CORPORATE）的产品，须在渠道端「用户注册组件 + 客户管理组件」完成注册与基础认证后，才允许调用授信申请/贷款申请/提额等后续组件；2.1 表「申请前是否要求注册登录」对公产品均为「是」。

若某产品需细分子流程（如不同渠道、不同担保方式），可在上述配置表中通过「产品编码+子类型」或扩展列区分，并在场景视图中增加对应列或子表。
