# 长沙银行-税E贷 — 进件端到端示例（授信申请场景）

[进件管理组件详细设计](../5.详细设计-单组件设计/组件设计-单组件样板-进件管理组件.md)

> 目标：用一条「税E贷对公授信申请 → 进件接收（授信进件、早同步）→ 股东会决议 → 办贷三合一/审批 → 签授信合同 → 额度激活 → 提款用信」的完整链路，串起**渠道端组件 → 进件管理组件（createIntake/updateIntakeStatus）→ 早同步策略/状态机 → 办贷端工作流与业务组件**，作为《进件管理组件 — 单组件详细设计》的配套示例。本示例为**授信进件（CREDIT_APPLY）**、**对公独立授权**、**授信与用信分离**，与公积金贷款申请（LOAN_APPLY）形成对照。

---

## 一、整体链路鸟瞰

以「对公客户完成注册认证与独立授权后发起税E贷授信申请，进件接收后走办贷三合一/审批、签授信合同、额度激活、再提款用信」为主线（场景 A：授信申请组件 → 进件管理，`intakeType=CREDIT_APPLY`）：

1. **预授信（可选）**：办贷端或外部通过**预授信组件**维护/推送白名单，**风控组件**执行规则，**征信与外部数据管理组件**提供数据源。  
2. **对公客户注册认证**（必经）：渠道前端 **用户注册组件** → **客户管理组件**、系统管理/消息/人脸/安全；**对公须先完成注册与基础认证**后方可进入授权与申请。  
3. **查询产品信息**：**营销方案管理组件** → 产品管理、利率与费用、**预授信额度**、额度管理、合同管理（若存在预授信白名单/预授信结果，需要在本步带出可用的预授信额度/额度提示）。  
4. **客户授权准入（独立阶段）**：**客户授权准入组件** → 授权配置管理、合同管理（授权协议展示）、消息 → **签约组件**（授权签署）→ **风控组件**（授权准入规则）；**先授权后授信申请**，授权通过后方可进入授信申请页。  
5. **客户发起授信申请**：**授信申请组件**（渠道端）初始化授信申请页（营销方案管理组件提供要素配置）、填写授信信息，提交后调用**进件管理组件的进件接收接口（createIntake，`intakeType=CREDIT_APPLY`）**，创建授信进件，进件状态 = `RECEIVED`。  
6. **股东会决议**（对公）：企业负责人通过**股东会决议组件**发起股东会授权决议，**征信与外部数据管理组件**（工商/司法）提供股东名单，**签约组件**按股东成员生成决议签署任务，所有股东成员决议完成后可与进件接收并行或按产品顺序要求配合。  
7. **系统进件接收**：进件管理在 `RECEIVED` 下按**早同步**策略对客户/评级/授信/用信等模块执行初始化；从授权配置获取进件规则包 ID，调用风控执行**进件准入**；税E贷**无额度阈值直通**，全部进件**进件自动分配**至客户经理队列，由工作流启动办贷流程。  
8. **客户经理补录** → **利率/额度调整与额度激活**（办贷端）：客户管理组件补录；授信管理/额度管理组件与工作流完成利率调整、额度调整、**额度激活**（税E贷为显式额度激活，与用信分离）。  
9. **三合一与审批**：授信管理组件牵头评级/授信/用信/风控 → 工作流/审批流 → 审批通过后**合同管理组件**生成授信合同、**签约组件**生成签约任务。  
10. **客户在线签授信合同**：签约组件完成签署后回调**进件管理 updateIntakeStatus（`newStatus = SIGNED`）**；进件管理在 **SIGNED** 下按同步策略对客户/授信/用信/评级/合同执行更新或终态确认（早同步下多为更新）。  
11. **客户发起提款（用信）**：放款组件（渠道）→ 用信管理/额度管理/合同管理 → 安全/人脸/消息 → **签约组件**（放款申请书）→ 放款组件（办贷）→ 核心 → 进件管理 **updateIntakeStatus（`newStatus = DISBURSED`）**。  
12. **受托支付审批**（按配置）、**还款**：同他产品。

## 二、关键节点与组件角色

### 2.1 渠道端：对公注册、独立授权、授信申请

- **用户注册组件 / 客户管理组件**：对公客户**必须先**完成注册与基础认证，方可进入授权与授信申请。  
- **客户授权准入组件（独立阶段）**：在授信申请**之前**独立完成授权准入；授权配置管理提供「业务阶段=授权准入」的授权文件与规则；签约组件完成授权签署后，风控组件执行授权准入规则，通过后方可进入授信申请。  
- **授信申请组件（渠道端）**：承载对公客户发起税E贷**授信申请**（场景 A，`CREDIT_APPLY`）；前置条件包括客户可识别、**先授权后申请**（独立授权阶段已完成）；提交后调用进件管理 **createIntake**，`intakeType=CREDIT_APPLY`。

### 2.2 进件接收：createIntake（CREDIT_APPLY）

- 上游：**授信申请组件（渠道端）**  
  - 构造 `CreateIntakeRequest`：`productId`（税E贷）、`intakeType=CREDIT_APPLY`、`channel`/`sourceSystem`、`customerId` 或 `customerSnapshot`、`customerType=CORPORATE`、授信申请业务字段（金额、期限、用途等）、`authRecordId`（独立授权阶段产生的授权记录）等。

- 进件管理组件内部处理：  
  1. 幂等检查；校验必填与产品/进件类型（CREDIT_APPLY）合法性；对公客户须已注册与基础认证（由调用方保证）。  
  2. 生成 `intakeId`，写入进件表，状态 = `RECEIVED`。  
  3. **早同步**：在 `RECEIVED` 下对客户/评级/授信/用信等模块执行初始化 `upsertByIntake`。  
  4. 授权配置获取进件阶段规则包 ID；若配置要求先授权则校验授权记录；调用风控执行**进件准入**。  
  5. 税E贷配置为**全部转线下**（无 R1 直通）：进件自动分配至客户经理队列，调用工作流启动办贷流程模板（客户经理补录 → 三合一/审批等）。

### 2.3 股东会决议与办贷端

- **股东会决议组件**：对公特有；征信与外部数据管理（工商/司法）提供股东名单，签约组件按股东名单生成股东会决议签署任务，与进件接收、办贷流程按产品顺序配合。  
- **客户经理补录、利率/额度调整、额度激活**：客户管理组件补录；授信管理/额度管理组件与工作流完成额度激活（税E贷**额度激活必选**，与用信分离）。  
- **三合一与审批**：授信管理 → 评级/授信/用信/风控 → 工作流/审批流 → 审批通过后合同管理生成**授信合同**、签约组件生成签约任务（**授信合同**，非借款合同）。

### 2.4 签约与状态更新：updateIntakeStatus（SIGNED / DISBURSED）

- **签授信合同**：签约组件完成授信合同签署后，回调进件管理 **updateIntakeStatus**：`newStatus = SIGNED`。进件管理在 SIGNED 下按同步策略对客户/授信/用信/评级/合同执行更新或终态确认。  
- **提款（用信）**：客户在额度项下发起提款，放款组件办贷、核心放款成功后，回调进件管理 **updateIntakeStatus**：`newStatus = DISBURSED`。

## 三、与产品场景、配置及同步策略的对应关系

- **场景视图**：  
  - 《产品流程-组件调用场景视图与配置表》**1.4 长沙银行-税E贷**：阶段 1～4 预授信、对公注册、查询产品、**独立授权准入**；阶段 5 客户发起**授信申请** → 进件管理（授信进件）；阶段 6 股东会决议 + 系统进件接收（早同步、授信进件、进件分配）；阶段 7～13 客户经理补录、利率/额度/额度激活、三合一与审批、客户在线签**授信合同**；阶段 14～16 提款、受托支付、还款。

- **产品配置表**：  
  - 产品编号 **CSB-TAX**（长沙银行-税E贷），客户类型 **CORPORATE**；**授信进件**，贷款/用信为额度项下提款；**早同步**；**多轮签约**（授信合同 → 放款申请书）；**额度激活必选**；对公先注册、授信与用信分离。

- **同步策略（早同步）**：  
  - 《进件与同步策略配置表》：CSB-TAX 授信进件在 **RECEIVED** 触发客户/评级/授信/用信初始化；**SIGNED**（授信合同签约后）触发同上模块更新/终态确认。

- **进件类型与授权**：  
  - 本示例为 **CREDIT_APPLY**，与《进件管理组件 — 单组件详细设计》场景 A（授信申请组件 → 进件管理）一致；**独立授权阶段**（AUTHORIZATION）在授信申请前完成，授权配置见 2.3 授权配置表 CSB-TAX / CORPORATE / AUTHORIZATION。

## 四、与公积金贷款示例的对照（小结）

| 维度 | 长沙银行-税E贷 | 湖南农信-公积金贷款 |
|------|----------------|----------------------|
| 客户类型 | **对公（CORPORATE）**，须先注册与基础认证 | 个人（PERSONAL），可临时客户进件 |
| 授权 | **独立授权阶段**，先授权后授信申请 | 申请内授权，在贷款申请页内完成 |
| 进件类型 | **CREDIT_APPLY（授信进件）** | LOAN_APPLY（贷款进件） |
| 同步时机 | **早同步**（RECEIVED 初始化，SIGNED 更新） | 早同步 |
| 路由 | 全部转线下，无额度阈值直通 | 额度≤20W 直通签约，>20W 转线下 |
| 签约 | **多轮**：先**授信合同**，后提款时**放款申请书** | 一轮：借款合同 |
| 额度与用信 | **显式额度激活**，激活后用信（提款） | 无单独额度激活阶段（视产品） |

本示例与公积金、湖北农信等端到端示例共用同一套进件管理接口与状态机，差异由**进件类型（CREDIT_APPLY vs LOAN_APPLY）**、**授权配置（独立授权 vs 申请内授权）**、**同步策略表**与**产品/客户类型/签约轮次与额度激活配置**承载，便于扩展政采贷等其他对公/授信进件产品的端到端示例。

## 五、与 CSV 对齐的补充场景（预授信与提款）

> 本节补充 `产品业务流程与组件综合分析.csv` 中税E贷在**预授信阶段**与**提款阶段**的细节，确保端到端示例覆盖 CSV 中的关键处理逻辑与说明。

### 5.1 （可选）预授信与白名单维护

- **对应 CSV 行**：税E贷中关于预授信/白名单由预授信组件 + 风控组件 + 征信与外部数据管理组件处理的阶段  
- **前置条件要点**：
  - 已配置税E贷预授信规则；白名单来源（客户经理维护或外部平台推送）可用；
  - 征信与外部数据管理组件可访问工商/税务/征信等外部数据源。
- **进入组件与内部处理逻辑**：
  - 进入组件：**预授信组件**；
  - 内部调用链：  
    `预授信组件 → 风控组件（预授信规则包/白名单校验）→ 征信与外部数据管理组件（工商/税务/征信等数据）`
- **关键说明**：
  - 预授信结果（预授信额度、白名单标记）不会直接落为正式授信，仅作为后续查询产品信息与授信申请的参考；
  - 在本端到端示例中，这一阶段可视为**在授信申请前的可选前置步骤**，不改变「独立授权 → 授信申请 → 授信进件 → 早同步 → 办贷审批」主线。

### 5.2 阶段 14：客户发起提款申请

- **对应 CSV 行**：营销产品「长沙银行-税E贷」、业务阶段「14、客户发起提款申请」  
- **前置条件要点**：
  - 客户已签署必要合同/放款申请书（视产品而定）；
  - 用信管理组件中存在可提款的用信/合同记录；
  - 额度管理组件中有可用额度；
  - 系统管理中已配置提款前安全验证策略（短信/人脸/密码等）。
- **进入组件与内部处理逻辑**：
  - 前置入口：**渠道前端**；
  - 进入组件：**放款组件（渠道端）**；
  - 内部调用链（与 CSV 一致）：  
    `放款组件（渠道）→ 用信管理/额度管理/合同管理组件 → 安全/人脸/消息 → 签约组件（放款申请书）→ 放款组件（办贷）→ 核心系统 → 进件管理组件`
- **关键说明（来自 CSV「说明」列）**：
  - **一个授信合同项下支持多次提款**；
  - 提款前风控/验证按产品配置，可在放款组件中按规则包/配置触发风控组件与安全组件；
  - 提款完成后需**更新进件状态**（通常通过放款组件办贷端回调 `updateIntakeStatus(DISBURSED)` 实现），与本示例 2.4 节的描述保持一致。
