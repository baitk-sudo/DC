# 湖北农信-公积金贷款（线上）— 进件端到端示例

[进件管理组件详细设计](../5.详细设计-单组件设计/组件设计-单组件样板-进件管理组件.md)

> 目标：用一条「湖北公积金贷款申请 → 风控 R1 直通 → 客户在线签借款合同 → 晚同步 → 提款/放款」的完整链路，串起**渠道端组件 → 进件管理组件（createIntake/updateIntakeStatus）→ 晚同步策略/状态机**，作为《进件管理组件 — 单组件详细设计》的配套示例。本示例走 **R1（直通签约）** 分支，无客户经理三合一/审批环节，与《湖北农信-公积金贷款-进件端到端示例-线上转线下》的 R3 分支形成对照。

---

## 一、整体链路鸟瞰

以「客户在渠道前端发起湖北公积金贷款申请，风控结果为 R1（通过），直通在线签约」为主线（场景 B：贷款申请组件 → 进件管理，`intakeType=LOAN_APPLY`；R1 分支）：

1. 客户在渠道前端完成**注册认证**（用户注册组件 → 客户管理组件临时客户），**查询产品信息**（营销方案管理组件 → 产品管理、利率、额度、合同等）。  
2. 客户在**贷款申请组件**中填写贷款申请信息；可选调用**申请前规则包**（风控组件）、**客户管理组件**补录；**申请内授权**（客户授权准入组件 → 授权配置/合同管理/签约组件签署授权协议）。提交后调用**进件管理组件的进件接收接口（createIntake，`intakeType=LOAN_APPLY`）**，创建贷款进件，进件状态 = `RECEIVED`。  
3. 进件管理组件按**晚同步**策略：在 `RECEIVED` 下**不**对客户/评级/授信/用信/额度等业务模块执行初始化，仅进件落库；从**授权配置管理组件**获取进件阶段规则包 ID，调用风控组件执行**进件准入**，得到 **R1（通过）**。  
4. 进件管理按路由配置走 **R1 分支**：调用**签约组件**生成签约任务，进件自动分配至「在线签约」或客户自助任务池，**不**启动办贷工作流、无客户经理三合一/审批。  
5. 客户在渠道/进件端**在线签借款合同**：签约组件 → 合同管理（可展示内容）→ 消息/人脸/安全 → 合同管理（签署回写）→ 签约完成时，**签约组件**回调**进件管理 updateIntakeStatus（`newStatus = SIGNED`）**；进件管理在 **SIGNED** 下按晚同步策略**一次性**调用客户/用信/授信/评级/合同管理组件 upsertByIntake，完成「申请 → 正式业务」落地。  
6. 客户**发起提款**：放款组件（渠道）→ 用信/额度/合同管理 → 安全/人脸/消息 → 签约组件（放款申请书）→ 放款组件（办贷）→ 核心系统 → 进件管理 **updateIntakeStatus（`newStatus = DISBURSED`）**。  
7. 若产品配置启用**受托支付**，办贷/工作流对放款进行审批；**还款**由还款组件（渠道→办贷）对接用信管理/利率与费用/核心/贷后/监管。

## 二、关键节点与组件角色

### 2.1 渠道端：注册、产品信息、贷款申请

- **用户注册组件**：客户注册、认证操作；**客户管理组件**：临时客户创建。  
- **营销方案管理组件**：聚合产品列表/详情、利率优惠券、额度可用额、合同可用额等查询，并通过**预授信组件**展示预计可贷款金额，通过**产品管理组件**展示产品最高可贷款金额；本阶段仅负责「看清产品与额度」，不做黑/白名单拦截。  
- **贷款申请组件**：  
  - 初始化贷款申请页面（服务）、填写贷款申请信息；  
  - 可选：风控**申请前规则包**（反欺诈、黑/白名单、产品参数准入，按产品配置）；客户管理补录；  
  - **申请内授权**：客户授权准入组件 → 授权配置管理、合同管理（授权协议展示）、消息/人脸 → 签约组件（授权签署）→ 提交前完成授权；  
  - 提交前若配置了申请前规则包且校验未通过，应在本阶段直接拦截（不创建进件，也不进入授权/进件准入）；通过后才调用进件管理 **createIntake**，传入 requestId、productId、intakeType=LOAN_APPLY、channel、customerId 或 customerSnapshot、业务字段、authRecordId 等。

### 2.2 进件接收：createIntake（LOAN_APPLY）与 R1 直通

- 进件管理组件：  
  1. 幂等检查；校验必填与产品/进件类型；生成 `intakeId`，状态 = `RECEIVED`。  
  2. **晚同步**：湖北模式在 RECEIVED 下不触发业务模块写入。  
  3. 授权配置管理组件（进件阶段规则包 ID）→ 客户管理（补录，若需）→ 风控组件（进件准入）。  
  4. 路由：**R1** → **签约组件**生成签约任务，进件分配至在线签约；拒绝 → `REJECTED`；R3 → 转线下（见线上转线下示例）。

### 2.3 签约与晚同步：updateIntakeStatus（SIGNED）

- **客户在线签借款合同**：签约组件展示合同、消息/人脸/安全校验、合同管理签署回写；签约完成时调用进件管理 **updateIntakeStatus**：`newStatus = SIGNED`。  
- 进件管理：更新进件状态为 SIGNED；查同步策略表，在 **SIGNED** 下对客户、评级、授信、用信、额度、合同执行 **upsertByIntake**（新建/更新正式客户、授信、用信、评级、合同终态确认），实现晚同步「合同签订完成后一次性写业务表」。

### 2.4 提款与放款：updateIntakeStatus（DISBURSED）

- 放款组件（渠道）发起提款 → 用信/额度/合同管理、安全/人脸/消息、签约（放款申请书）→ 放款组件（办贷）→ 核心系统放款成功 → 回调进件管理 **updateIntakeStatus**：`newStatus = DISBURSED`。  
- 进件管理更新状态，按策略执行 DISBURSED 下终态或报表动作。

## 四、与线上转线下示例的对照（小结）

| 维度 | 湖北农信-公积金（线上） | 湖北农信-公积金（线上转线下） |
|------|--------------------------|--------------------------------|
| 风控结果分支 | **R1**：直通签约，无办贷人工环节 | **R3**：转线下，客户经理受理、三合一、审批后再签约 |
| 阶段 4 后 | 直接生成签约任务 → 客户签约 → 晚同步 | 进件分配 → 客户经理个人客户创建 → 三合一 → 审批 → 生成合同与签约任务 → 客户签约 → 晚同步 |
| 进件状态流转 | RECEIVED →（R1）→ 签约任务 → SIGNED → DISBURSED | RECEIVED →（R3）→ 分配 → UNDER_REVIEW/APPROVED（办贷）→ SIGNED → DISBURSED |
| 同步策略 | 同为**晚同步**，SIGNED 时一次性写业务表 | 同上 |

同一产品「线上」与「线上转线下」共用同一套 createIntake、晚同步策略与 updateIntakeStatus，仅由**风控准入结果（R1/R3）**与**路由配置**决定是否进入办贷流程，便于与《湖北农信-公积金贷款-进件端到端示例-线上转线下》及《湖南农信-公积金贷款-进件端到端示例》对照阅读。

## 五、与 CSV 对齐的补充场景（提额）

> 本节补充 `产品业务流程与组件综合分析.csv` 中本产品的**可选提额阶段**，用于在需要时作为端到端扩展阅读；主干实现仍以前文「贷款申请 → 进件 → 签约 → 提款」为核心。

### 5.1 （可选）客户发起提额申请

- **对应 CSV 行**：营销产品「湖北农信-公积金贷款（线上）」、业务阶段「（可选）客户发起提额申请」  
- **前置条件要点**：
  - 客户已在渠道端完成注册认证，具备可识别的客户标识；
  - 客户已存在**前置的贷款申请/贷款进件**；
  - 营销方案（产品）管理组件中已配置提额申请要素与规则；如产品要求先预授信/先授权，需要能通过预授信组件或授权准入组件验证对应前置条件，否则可按“直接提额”模式进入进件。
- **进入组件与内部处理逻辑**（对齐 CSV）：
  - 前置入口：**营销方案（产品）管理组件**（为提额申请页提供要素配置与初始化服务）；
  - 进入组件：**提额组件（渠道端）**；
  - 内部调用链：  
    `提额组件 → 营销方案管理组件（申请页初始化）→ 风控组件（申请前规则包：反欺诈、黑/白名单、产品参数准入，按产品配置）→ 消息/人脸 → 签约组件（授权）→ 进件管理组件`
- **关键说明（来自 CSV「说明」列）**：
  - 客户需先完成注册认证，且**需有前置的贷款申请**；
  - 提交前按产品配置执行**申请前规则包**：未通过则在本阶段直接拦截（不创建进件、也不进入授权/进件准入）；
  - 通过后才进入授权准入与后续进件；授权协议展示由合同管理提供；模式为**申请内授权**。

### 5.2 （可选）提额进件接收

- **对应 CSV 行**：业务阶段「（可选）提额进件接收」  
- **前置条件要点**：
  - 传入的提额申请/进件数据完整（至少包含产品、进件类型、客户标识、基本业务字段）；
  - 对应产品在同步策略表中已有配置，进件状态机定义完整；
  - 授权配置中若要求「先授权」，则在进件创建或进入关键状态前，可基于授权配置管理组件 + 授权记录校验该前置条件；
  - **必须存在前置的授信申请或贷款申请**，以便校验提额目标对象与额度。
- **进入组件与内部处理逻辑**：
  - 前置入口：**提额组件（渠道端）**；
  - 进入组件：**进件管理组件**；
  - 内部调用链：  
    `进件管理组件 → 授权配置管理组件（查询进件阶段规则包 ID）→ 客户管理组件 → 风控组件（准入，规则包由授权配置提供）→ [R1] 签约组件（生成签约任务）/ [R3] 进件管理组件（转线下，按配置办贷任务类型）`
- **关键说明（来自 CSV「说明」列）**：
  - **晚同步**：提额进件环节至落地进件信息，不同步/评级/授信/用信信息，与贷款申请进件保持一致；
  - 按业务环阶段「产品+贷款进件阶段」查询当前需要执行的规则包；
  - R1 分支生成签约任务，R3 分支可按配置转线下，由三合一/二合一等流程处理提额审批。
