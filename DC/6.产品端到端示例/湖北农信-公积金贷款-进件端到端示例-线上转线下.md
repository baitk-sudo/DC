# 湖北农信-公积金贷款（线上转线下）— 进件端到端示例

[进件管理组件详细设计](../5.详细设计-单组件设计/组件设计-单组件样板-进件管理组件.md)

> 目标：用一条「湖北公积金贷款申请 → R3 转线下办贷 → 三合一/审批 → 签约 → 放款」的完整链路，串起**渠道端组件 → 进件管理组件（createIntake/updateIntakeStatus）→ 晚同步策略/状态机 → 办贷端工作流与业务组件**，作为《进件管理组件 — 单组件详细设计》的配套示例；与湖南公积金示例对比可看出**晚同步**与**早同步**、**R3 转线下办贷任务类型**等差异。

---

## 一、整体链路鸟瞰

以「客户在渠道前端发起湖北公积金贷款申请，风控结果为 R3 转人工，进入线上转线下办贷」为主线（场景 B：贷款申请组件 → 进件管理，`LOAN_APPLY`；R3 分支）：

1. 客户在渠道前端完成**注册认证**（用户注册组件 → 客户管理组件临时客户），**查询产品信息**（营销方案管理组件）；在**贷款申请组件**中填写贷款申请信息；**可选**调用**申请前规则包**（风控组件）、**客户管理组件**补录；**申请内授权**（客户授权准入组件 → 授权配置/合同管理/签约组件签署授权协议）。提交后调用**进件管理组件的进件接收接口（createIntake，`intakeType=LOAN_APPLY`）**，创建贷款进件，进件状态 = `RECEIVED`。  
2. 进件管理组件按**晚同步**策略：在 `RECEIVED` 状态下**不**对客户/评级/授信/用信/额度等业务模块执行初始化，仅做进件落库与风控准入。（阶段 1～3 与线上模式一致，含可选申请前规则包、申请内授权。）  
3. 进件管理组件从**授权配置管理组件**获取进件阶段规则包 ID，调用风控组件执行**进件准入**，得到 R1/R3/拒绝。本示例走 **R3（转人工）**。  
4. 进件管理组件按路由配置解析**办贷任务类型**（如三合一或二合一(授信+用信)，由产品+进件类型+渠道配置），调用**工作流/审批流组件**启动对应办贷流程模板，进件自动分配至客户经理队列。  
5. 客户经理在办贷端**手工受理**进件，触发**客户管理组件**补录客户信息（阶段 5）；再办理**三合一任务**（阶段 6）：评级管理 → 授信管理 → 用信管理 → 风控 → 授信管理提交，由**工作流/审批流组件**流转至有权审批人。  
6. 审批通过后，**工作流/审批流**驱动**合同管理组件**生成合同信息、**签约组件**生成签约任务；办贷端或渠道**客户在线签借款合同**。  
7. 签约完成时，**签约组件**回调**进件管理组件的 updateIntakeStatus（`newStatus = SIGNED`）**；进件管理按**晚同步策略**在 `SIGNED` 状态下**一次性**调用客户/评级/授信/用信/额度/合同等模块的 upsertByIntake，完成「申请 → 正式业务」的落地。  
8. 后续提款、受托支付（按配置）、放款、还款与线上模式一致；放款完成时放款组件回调 `updateIntakeStatus（newStatus = DISBURSED）`，进件管理更新状态并按策略执行终态或报表类动作。

## 二、关键节点与组件角色

### 2.1 渠道端：贷款申请前置（与线上模式一致）

- **营销方案管理组件（渠道前端）**  
  - 提供湖北公积金贷款产品列表/详情及贷款申请页面要素配置。

- **贷款申请组件（渠道端）**  
  - 职责：承载客户在渠道上发起湖北公积金贷款申请（场景 B，`LOAN_APPLY`）。  
  - 关键前置：客户识别、申请内授权（授权配置决定授权文件与规则包）。  
  - **可选**：风控**申请前规则包**（反欺诈、黑名单、产品参数准入，按产品配置）；客户管理补录。  
  - 提交后调用进件管理 **createIntake**，与线上模式相同。

### 2.2 进件接收：createIntake（LOAN_APPLY）与 R3 转线下

- 上游：**贷款申请组件（渠道端）**  
  - 构造 `CreateIntakeRequest`（requestId、productId、intakeType=LOAN_APPLY、channel、customerId 或 customerSnapshot、业务字段、authRecordId 等），与湖南示例类似。

- 进件管理组件内部处理：  
  1. 幂等检查；校验必填与产品/进件类型合法性；生成 `intakeId`，写入进件表，状态 = `RECEIVED`。  
  2. **晚同步**：查同步策略表，湖北模式在 `RECEIVED` 下**不**触发客户/评级/授信/用信/额度/合同的写入，进件仅作为「申请暂存 + 审批过程载体」。  
  3. 调用授权配置管理组件获取**进件阶段规则包 ID**，调用风控组件执行**进件准入**。  
  4. 按路由配置对风控结果处理：  
     - 拒绝：进件状态置为 `REJECTED`；  
     - **R1**：生成签约任务，客户可在线直通签约（与线上模式一致）；  
     - **R3**：解析**办贷任务类型**（三合一/二合一(授信+用信)等，由产品+进件类型+渠道「线上转线下」配置），调用工作流组件启动对应办贷流程模板，进件自动分配至客户经理队列。

### 2.3 办贷端：客户经理受理、三合一、审批

- **阶段 5：客户经理个人客户创建**  
  - 客户管理组件：补录/修正客户信息，持久化后供三合一与风控使用。

- **阶段 6：客户经理线下三合一**  
  - 授信管理组件牵头：评级管理（评级信息填写）→ 授信管理（授信信息、额度测算）→ 用信管理（用信信息）→ 风控组件（客户风控准入）→ 授信管理提交 → 工作流/审批流（流转至审批节点）。

- **阶段 7：有权审批人业务审查审批**  
  - 工作流/审批流组件执行审批动作；审批通过后，授信管理写入审批结论，**合同管理组件**生成合同信息，**签约组件**生成签约任务。  
  - 此时可回调进件管理 **updateIntakeStatus**：`currentStatus=UNDER_REVIEW`，`newStatus=APPROVED`，`sourceSystem=WORKFLOW`；湖北晚同步模式下，**APPROVED 可不配置业务模块写入**（或仅做最小化记录），正式落地在 SIGNED。

### 2.4 签约与晚同步：updateIntakeStatus（SIGNED）

- **客户在线签借款合同**  
  - 签约组件完成签署后，调用**进件管理 updateIntakeStatus**：`newStatus = SIGNED`，`sourceSystem = CONTRACT/SIGN`。  
  - 进件管理：  
    - 校验状态迁移合法性，更新进件状态为 `SIGNED`；  
    - 查同步策略表，湖北模式在 **SIGNED** 下对客户、评级、授信、用信、额度、合同执行 **upsertByIntake**（新建/更新正式客户、授信、用信、评级、合同终态确认），实现「合同签订完成后一次性同步到业务模块」。

### 2.5 提款与放款：updateIntakeStatus（DISBURSED）

- 客户发起提款、放款组件办贷、核心放款成功后，**放款组件**回调进件管理 **updateIntakeStatus**：`newStatus = DISBURSED`。  
- 进件管理更新进件状态，若策略表对 DISBURSED 有配置则执行终态确认或报表触发。

## 三、与产品场景、配置及同步策略的对应关系

- **场景视图**：  
  - 在《产品流程-组件调用场景视图与配置表》**1.2 湖北农信-公积金贷款（线上转线下）**中，**阶段 1～3 与线上一致**（含可选**申请前规则包**、客户管理补录、申请内授权）；阶段 4 进件接收后 R3 转线下，阶段 5～7 为办贷端客户经理个人客户创建、三合一、审批，5' 为客户在线签借款合同并触发晚同步，6'～8 为提款/受托支付/还款。

- **风控与路由配置（申请前规则包）**：  
  - 《产品流程-组件调用场景视图与配置表》**2.4 风控与路由配置表**中，**HBGJJ-O2O** 与 **HBGJJ-ONLINE** 均配置「申请前（可选）」：申请前规则包（反欺诈、黑名单、产品参数准入），规则包来源为授权配置或产品配置，按产品配置是否启用；未通过不可提交进件。线上转线下与线上在阶段 3 的规则一致，实施时需保证两产品在该配置上可复用或显式同步。

- **产品配置表**：  
  - 产品编号 **HBGJJ-O2O**（湖北农信-公积金-线上转线下），晚同步；R3 转线下办贷（办贷任务类型按配置）。

- **同步策略（晚同步）**：  
  - 与《进件中心-业务数据同步策略设计》5.1 湖北农信模式一致：客户、评级、授信、用信、额度、合同均在 **SIGNED** 触发同步（新建/更新/终态确认），RECEIVED～APPROVED 不写或仅最小化临时记录。

- **进件类型与办贷任务类型**：  
  - 本示例为 `LOAN_APPLY`；R3 分支下办贷任务类型（三合一/二合一(授信+用信)等）由**产品+进件类型+渠道（线上转线下）**配置，进件管理解析配置并驱动工作流启动对应流程模板，与《组件设计-第4段-进件接收与风控准入》及《限界上下文与集成关系-ContextMap》一致。

## 四、与湖南公积金示例的对比（小结）

| 维度 | 湖北农信-公积金（线上转线下） | 湖南农信-公积金 |
|------|------------------------------|------------------|
| 同步时机 | **晚同步**：SIGNED 时一次性写客户/评级/授信/用信/额度/合同 | **早同步**：RECEIVED 即初始化，APPROVED/SIGNED 更新或终态确认 |
| RECEIVED 时业务模块 | 不写入 | 初始化客户、评级、授信、用信、额度等 |
| R3 转线下 | 办贷任务类型（三合一/二合一）按配置，工作流启动对应模板 | 同上，三合一/二合一/单独按配置 |
| 签约时进件管理 | updateIntakeStatus(SIGNED) 触发晚同步一次性 upsertByIntake | updateIntakeStatus(SIGNED) 多为更新/终态确认（早同步已初始化） |

本示例与《湖南农信-公积金贷款-进件端到端示例》共用同一套进件管理接口与状态机，差异由**同步策略表**与**产品/进件类型/渠道路由与办贷任务类型配置**承载，便于扩展其他产品（如税E贷、政采贷）的端到端示例。
