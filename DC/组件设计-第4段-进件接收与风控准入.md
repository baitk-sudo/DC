# 组件设计分析 — 第 4 段：进件接收与风控准入

本段覆盖流程主线中**进件接收、风控准入、路由分支（R1 直通 / R3 转人工）、同步策略执行**以及**进件自动分配**等环节，与《进件中心-业务数据同步策略设计》紧密联动，确保同一套进件中心通过配置支持「晚同步」与「早同步」、直通与转人工等多种模式。

**本段涉及的组件**（办贷管理端 v2）：

- 进件管理组件
- 风控组件（本段侧重进件阶段风控准入与规则执行）
- 工作流/审批流组件
- 客户管理组件、评级管理组件、授信管理组件、用信管理组件、额度管理组件、合同管理组件、签约组件（作为同步策略的**被调用方**在本段出现）

**已对照产品**：湖北农信-公积金贷款（线上/线上转线下）、湖南农信-公积金贷款、长沙银行-税E贷、长沙银行-政采贷。

---

## 一、进件管理组件

### 1.1 职责与边界

| 项目 | 说明 |
|------|------|
| **职责** | 统一接收来自渠道的授信进件、贷款进件、提额进件；维护进件单生命周期与状态机（RECEIVED → UNDER_REVIEW → APPROVED → SIGNED → DISBURSED / REJECTED）；在状态迁移时按**同步策略配置表**触发对各业务模块的同步（客户、评级、授信、用信、额度、合同等）；根据风控准入结果与产品配置驱动路由（R1 直通 / R3 转人工、或按额度阈值等规则分支）；执行进件自动分配，与工作流组件协同生成待办任务（如签约任务、三合一任务）。 |
| **边界** | 不执行业务规则或额度测算逻辑，规则执行在风控组件；不定义审批流程节点与审批人，流程定义在工作流/审批流组件；不直接生成合同内容或签约任务内容，合同与签约由合同管理、签约组件提供；同步动作的「何时、对谁、做什么」由同步策略配置决定，进件管理只负责按配置调用各业务组件的 upsertByIntake 类接口。 |

### 1.2 多产品 / 多模式支持点

- **进件类型**：支持授信进件、贷款/用信进件、提额进件等类型，不同进件类型可配置不同的下游流程模版（直通签约、转人工三合一、仅人工审批等）。
- **同步时机配置化**：通过《进件中心-业务数据同步策略设计》中的策略表，按产品 + 进件状态（RECEIVED / APPROVED / SIGNED 等）配置对各业务模块的触发时机与动作（初始化/更新/终态确认），从而支持湖北「晚同步」与湖南/长沙「早同步」等差异。
- **路由规则配置化**：支持按产品配置「风控准入结果 + 额度阈值 + 其他条件」决定走直通（R1）还是转人工（R3）；例如湖南公积金「20W 以下直通、20W 以上转人工」通过配置实现，不写死。
- **分配规则**：进件自动分配时，可按机构、产品、客户经理负载、客户归属等配置分配策略，由工作流/审批流组件执行具体分配与任务生成。

### 1.3 关键接口与上下游关系

| 接口/能力 | 说明 | 上游/下游 |
|-----------|------|-----------|
| 进件接收 | 接收渠道提交的授信/贷款/提额申请数据，落库进件单，置状态为 RECEIVED，并触发 RECEIVED 对应的同步策略（若配置了早同步） | 下游：同步策略执行→客户/评级/授信/用信/额度等；上游：授信申请组件、贷款申请组件、提额组件 |
| 补充临时客户信息 | 在进件环节再次调用客户管理组件补全或更新客户信息 | 下游：客户管理组件；上游：进件管理流程 |
| 触发风控准入 | 调用风控组件执行进件阶段准入规则包，获取 R1/R3 等结果 | 下游：风控组件；上游：进件管理组件 |
| 根据准入结果路由 | 根据风控结果与产品路由配置，决定生成签约任务（直通）或转人工队列（三合一/审批） | 下游：工作流/审批流组件、签约组件、合同管理组件；上游：进件管理组件 |
| 进件自动分配 | 将进件挂接到工作流节点或客户经理队列，按分配规则执行 | 下游：工作流/审批流组件；上游：进件管理组件 |
| 状态迁移与同步策略执行 | 在 APPROVED、SIGNED、DISBURSED 等状态迁移时，查询同步策略表并调用各业务组件 upsertByIntake | 下游：客户、评级、授信、用信、额度、合同等组件；上游：工作流回调或内部流转 |
| 生成三合一任务（R3 分支） | 在转人工分支下，通过工作流生成「评级+授信+用信」组合任务，由客户经理办理 | 下游：工作流/审批流组件、授信管理组件（业务牵头）；上游：进件管理组件 |

### 1.4 对已分析产品的支撑检查

| 产品 | 本段相关步骤 | 说明 |
|------|--------------|------|
| 湖北农信-公积金-线上 | 进件接收→补充临时客户→风控准入→R1 生成签约任务 / R3 转人工→进件自动分配；晚同步，签约时才写客户/授信/用信/评级/合同 | 进件管理按晚同步策略在 RECEIVED 不触发业务模块写入，SIGNED 时触发；R1/R3 由风控结果驱动 |
| 湖北农信-公积金-线上转线下 | R3 分支下进件分配→客户经理受理→生成三合一任务；后续审批完成→生成合同与签约任务 | 三合一任务类型由工作流配置，进件管理驱动工作流生成任务并更新进件状态 |
| 湖南农信-公积金 | 进件接收→系统创建客户、风控、评级、额度测算→按额度判断：20W 以下直通签约，20W 以上转人工；早同步 | 进件管理在 RECEIVED 执行早同步策略；路由规则需支持「额度阈值」配置 |
| 长沙银行-税E贷 | 进件接收→创建客户、风控、评级、额度、初始化授信/用信→客户经理手工受理（无额度直通）；早同步 | 全部进件转人工，无额度阈值直通；早同步策略在 RECEIVED 触发 |
| 长沙银行-政采贷 | 同税E贷，授信申请一律转线下；早同步 | 同上 |

**小结**：进件管理组件是本段核心，通过状态机 + 同步策略表 + 路由配置，统一支撑多产品、多同步时机、多分支流程，避免为每家机构写死逻辑。

## 二、风控组件（本段：进件阶段风控准入）

### 2.1 本段职责与边界

| 项目 | 说明 |
|------|------|
| **本段职责** | 在进件接收后执行「进件阶段风控准入」规则包（反欺诈、黑名单、准入评分等），返回准入结果（如 R1-通过、R3-转人工、拒绝）；为额度阈值等路由条件提供额度测算或额度结果输入（与额度管理、评级等协作）；不决定「何时同步业务数据」，仅输出风控结论供进件管理做路由与后续流程驱动。 |
| **边界** | 规则包按产品/进件类型由授权配置或产品配置引用；外部数据通过征信与外部数据管理组件获取；不维护进件状态，不执行同步，仅被进件管理组件调用并消费结果。 |

### 2.2 多产品 / 多模式支持点

- **规则包配置**：不同产品、不同进件类型可配置不同的进件准入规则包；新产品通过配置接入，无需改代码。
- **结果类型**：支持多级结果（通过/转人工/拒绝/补充资料等），进件管理根据结果选择流程分支。
- **与额度/评级的协作**：若路由依赖「额度阈值」，风控组件可调用额度测算或输出额度相关结果，供进件管理做分支判断；评级结果可由评级管理组件在同步策略中写入，风控仅消费或作为规则输入。

### 2.3 关键接口与上下游关系（本段）

| 接口/能力 | 说明 | 上游/下游 |
|-----------|------|-----------|
| 执行进件风控准入 | 接收进件标识、产品、客户等信息及规则包 ID，执行规则并返回准入结果（R1/R3/拒绝等） | 下游：征信与外部数据管理组件（取数）；上游：进件管理组件 |
| 额度测算（若路由需要） | 部分产品按额度阈值路由时，需额度测算结果；可由风控调用额度逻辑或由进件管理先调评级/额度再调风控 | 下游：额度管理组件、评级管理组件；上游：进件管理组件 |

**小结**：风控在本段是「准入规则执行引擎」，进件管理根据其输出与路由配置决定直通或转人工，不在此做业务数据落库。

## 三、工作流/审批流组件（本段职责）

### 3.1 本段职责与边界

| 项目 | 说明 |
|------|------|
| **本段职责** | 定义并执行进件接收后的流程分支：直通分支（生成签约任务、进件分配至自动签约池等）、转人工分支（进件分配至客户经理队列、生成三合一任务、后续审批节点）；进件状态变更（如 UNDER_REVIEW、APPROVED）由工作流节点完成时回调进件管理更新状态；分配规则（按机构、产品、负载等）在工作流中配置并执行。 |
| **边界** | 不执行业务规则，不写入客户/授信/用信/额度等业务表；业务写入由进件管理按同步策略调用各业务组件完成；工作流只负责「流程走向、任务生成、节点完成时回调进件管理」。 |

### 3.2 多产品 / 多模式支持点

- **流程模版**：支持多套进件后流程模版（如「直通签约」「转人工三合一+审批」「仅人工审批」），按产品或进件类型绑定不同模版。
- **三合一任务类型**：将「评级+授信+用信」定义为一种组合任务类型，节点内由业务服务依次调用评级、授信、用信、额度、风控等组件，授信管理可作为业务牵头，工作流只负责任务下发与流转。
- **分配规则**：支持按机构、产品、客户经理负载、客户归属等配置分配逻辑，便于多机构、多产品复用。

### 3.3 关键接口与上下游关系（本段）

| 接口/能力 | 说明 | 上游/下游 |
|-----------|------|-----------|
| 启动进件后流程 | 根据进件类型、产品、风控结果选择流程模版并启动实例 | 下游：无；上游：进件管理组件 |
| 进件分配 | 将进件或任务分配至客户经理/队列，更新进件所属或任务所属 | 下游：无；上游：进件管理组件 |
| 生成签约任务 | 在直通分支下，创建待签合同任务并关联进件、合同、客户 | 下游：签约组件、合同管理组件；上游：工作流节点 |
| 生成三合一任务 | 在转人工分支下，生成组合任务并下发给客户经理 | 下游：授信管理、评级、用信、额度、风控（节点内调用）；上游：工作流节点 |
| 节点完成回调 | 审批通过等节点完成时，回调进件管理更新进件状态（如 APPROVED），进件管理再执行该状态下的同步策略 | 下游：进件管理组件；上游：工作流引擎 |

**小结**：工作流/审批流组件与进件管理协同，实现「路由 + 任务 + 状态」的配置化，业务数据写入仍由进件管理通过同步策略统一驱动。

## 四、同步策略执行与各业务组件的协作（本段视角）

进件管理在状态迁移时，根据《进件中心-业务数据同步策略设计》中的策略表，对以下组件发起调用（均为按进件号幂等的 upsert 或等效接口）：

| 业务模块 | 在本段的角色 | 说明 |
|----------|--------------|------|
| 客户管理组件 | 被同步方 | RECEIVED（早同步）时创建/初始化客户；SIGNED（晚同步）时创建/更新正式客户；进件环节也可补全临时客户信息 |
| 评级管理组件 | 被同步方 | RECEIVED（早同步）时写入评级结果；SIGNED（晚同步）时写入或更新 |
| 授信管理组件 | 被同步方 | RECEIVED 时初始化授信申请信息；APPROVED/SIGNED 时更新或终态确认 |
| 用信管理组件 | 被同步方 | RECEIVED 时初始化用信申请信息；APPROVED/SIGNED 时更新或终态确认 |
| 额度管理组件 | 被同步方 | RECEIVED 时写入额度测算或预占；APPROVED/SIGNED 时正式占用或更新 |
| 合同管理组件 | 被同步方 | APPROVED 时生成合同草稿/待签；SIGNED 时合同生效 |
| 签约组件 | 被调用方 | 直通分支下生成待签任务，与合同管理协同；任务可下发给**面客渠道**或**办贷端 PAD** 现场签约（签约组件为公共能力，统一支撑两种入口） |

上述模块均需提供「按进件号幂等」的接口，供进件管理在对应状态迁移时调用，具体策略配置见《进件中心-业务数据同步策略设计》。

## 五、本段组件协作关系小结

```text
[渠道 授信/贷款/提额申请]  -->  进件管理组件（进件接收）
                                      |
                                      v
                              落库进件单，状态 RECEIVED
                                      |
                                      v
                    同步策略执行（若早同步）--> 客户/评级/授信/用信/额度 等 upsertByIntake
                                      |
                                      v
                              风控组件（进件准入规则包）
                                      |
                          R1-通过 / R3-转人工 / 拒绝
                                      |
          +----------------------------+----------------------------+
          v                                                          v
  工作流：直通分支                                            工作流：转人工分支
  - 生成签约任务（签约+合同管理）                              - 进件自动分配
  - 进件分配至签约池                                          - 生成三合一任务（授信/评级/用信/额度/风控）
                                                              - 客户经理办理 → 审批节点 → 回调进件状态 APPROVED
          |                                                          |
          v                                                          v
  进件状态 SIGNED 时执行同步策略（晚同步：写客户/授信/用信/评级/合同/额度）
```

## 六、与前序段落及后续段落的衔接

- **与第 3.5 段（授信/贷款/提额申请）**：三个渠道组件提交的申请在本段统一由进件管理接收，形成授信进件、贷款进件、提额进件，并进入风控与路由。
- **与第 3 段（授权准入）**：进件前若需授权，由授权准入与授权配置管理完成；进件接收后风控准入可能再次校验授权有效性，但不重复授权流程。
- **与《进件中心-业务数据同步策略设计》**：本段进件管理的状态机与同步策略执行逻辑与该文档完全一致，实现晚同步/早同步及多阶段更新的配置化。
- **与后续段落**：R1 直通分支将进入「签约与合同管理」段落；R3 转人工分支将进入「三合一任务与审批」段落，再回到签约与放款。

本文件为「按业务办理过程顺序」组件分析的第 4 段；第 5 段可继续分析**三合一任务与审批（授信管理、用信管理、评级管理、额度管理、工作流）**及**签约与合同管理**。
