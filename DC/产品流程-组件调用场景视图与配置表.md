# 产品流程 × 组件调用场景视图与配置表

本文档将各产品的业务办理流程与《组件设计-组件调用关系总览》对齐，形成：**按产品的组件调用场景视图** 与 **实施时需维护的配置表**，便于按产品落地实现与配置管理。

---

## 一、组件调用场景视图（按产品）

以下按产品列出「业务阶段 → 主调组件 → 被调组件链」的调用关系，与总览中的前置条件一致；表中「主调」为流程入口或编排方，「被调链」为在该阶段内依次或并行调用的组件。

### 1.1 湖北农信-公积金贷款（线上）

| 业务阶段 | 主调/入口 | 组件调用链（主调→被调） | 说明 |
|----------|-----------|--------------------------|------|
| 1、客户注册 | 渠道前端 | 用户注册组件 → 客户管理组件（临时客户）、系统管理/消息/人脸/安全 | 先注册认证再申请；个人客户 |
| 2、查询产品信息 | 渠道前端 | 营销方案管理组件 → 产品管理、利率与费用、额度管理、合同管理 | 列表/详情/优惠券/可用额/最高可贷额 |
| 3、客户发起贷款申请 | 渠道前端 | 贷款申请组件 → 客户管理（补录）→ 客户授权准入组件 → 授权配置管理 → 合同模版 → 消息/人脸 → 签约组件（授权）→ 进件管理组件 | 申请内嵌授权（征信+公积金）；提交后进件 |
| 4、进件接收 | 进件管理组件 | 进件管理 → 客户管理（补录）→ 风控组件（准入）→ [R1]签约组件（生成签约任务）/ [R3]进件管理（转人工） | 晚同步：SIGNED 前不写客户/评级/授信/用信 |
| 5、客户在线签借款合同 | 渠道/进件 | 签约组件 → 合同模版 → 消息/人脸/安全 → 合同管理 → 客户/用信/授信/评级/合同管理（SIGNED 时同步）→ 进件管理（状态） | 签约完成触发晚同步一次性写业务表 |
| 6、客户发起提款 | 渠道前端 | 放款组件（渠道）→ 用信/额度/合同管理 → 安全/人脸/消息 → 签约（放款申请书）→ 放款组件（办贷）→ 核心系统 → 进件管理 | 提款前风控/验证按产品配置 |

## 三、使用说明

- **组件调用场景视图**：按产品、按业务阶段查看「谁在什么环节调用谁」，与《组件设计-组件调用关系总览》中的前置条件一致；实现时需保证调用顺序与前置条件满足关系一致。
- **配置表**：实施时在系统中维护对应配置（或通过参数表/配置中心落地）；新增产品时在 2.1～2.5 各表中增加一行或若干行，并同步更新《产品规则-回归用例矩阵》与本文档的视图。
- **对公统一前置**：客户类型为对公（CORPORATE）的产品，须在「用户注册组件 + 客户管理组件」完成注册与基础认证后，才允许调用授信申请/贷款申请/提额组件；2.1 表「申请前是否要求注册登录」对公产品均为「是」。

若某产品需细分子流程（如不同渠道、不同担保方式），可在上述配置表中通过「产品编码+子类型」或扩展列区分，并在场景视图中增加对应列或子表。
