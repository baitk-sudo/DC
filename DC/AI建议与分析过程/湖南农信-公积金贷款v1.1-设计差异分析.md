# 湖南农信-公积金贷款 v1.1 CSV 与现有设计差异分析

> 基于《湖南农信-公积金贷款v1.1.csv》（现存放于 **产品业务流程/**）与现有设计文档（《产品流程-组件调用场景视图与配置表》《产品流程-业务步骤级组件分析》）的对照，梳理**命名差异、步骤差异、组件映射与设计建议**。

---

## 一、CSV v1.1 与设计文档的组件命名对应

| CSV v1.1 中的名称 | 设计文档/办贷 v2 对应组件 | 说明 |
|-------------------|---------------------------|------|
| 智能决策组件 | **风控组件**（规则引擎）+ 可选 **额度管理组件**（额度测算） | 准入规则、额度测算、签约前/提款前风控均映射为风控组件 |
| 电子签约平台 | **签约组件** | 授权协议签署、借款合同签署、放款申请书签署统一为签约组件 |
| 合同（协议）模版管理组件 | **合同（协议）模版管理组件** | 一致；设计约定签约展示内容由合同管理提供，签约不直接调模版 |
| 利率组件 | **利率与费用组件** | 一致 |
| 消息中心组件 | **消息中心组件** | 一致 |
| 人脸识别组件 | **人脸识别服务**（通用能力） | 一致 |
| 核心系统 | **核心/账务系统**（支付与账务集成） | 一致 |
| 安全组件 | **安全组件**（通用能力） | 一致 |

## 二、v1.1 与旧版 CSV 的步骤差异（湖南农信）

| 阶段 | v1.1 独有或变化 | 旧版 CSV | 设计建议 |
|------|-----------------|----------|----------|
| **1、预授信申请** | 步骤1「客户经理维护预产品授信白名」被调为空；步骤2「调用模式测算出预期授信额度」→ 智能决策组件 | 场景1/场景2 均为 智能决策组件 | 预授信阶段「白名单维护」可由预授信组件自身或批处理完成；「额度测算」由风控组件（或预授信规则）执行，设计已覆盖。建议 v1.1 步骤1 被调可填「预授信组件」或留空（主责在预授信）。 |
| **3、客户发起贷款申请** | 新增「调用风控中心执行规则包，（业务准入规则包：反欺诈、黑名单、产品参数准入）」→ 智能决策组件 | 无此步骤 | **设计差异**：当前设计为「申请提交后进件，进件接收阶段再执行风控准入」。若业务要求在**提交进件前**在渠道侧执行一次业务准入（反欺诈、黑名单、产品参数），需在设计中明确：① 贷款申请组件在提交前调用风控组件执行「申请前规则包」，或 ② 进件落库后立即触发同一规则包（当前进件接收已含风控）。建议采用 ② 与现有「进件→风控」一致；若产品强需求 ①，需在授权/申请段增加「提交前风控」配置与调用链。 |
| **3、客户发起贷款申请** | 「查看需要授权的文件」→ 授权配置管理组件；「加载查看协议文件」→ 合同（协议）模版管理组件；「签订授权协议」→ 电子签约平台 | 同左（签约组件） | 设计已覆盖；签约展示按设计由合同管理提供，协议展示可由授权配置+合同管理（或模版）提供。 |
| **4、系统进件接收** | 「查询授权配置管理确认需要执行的风控规则」→ 授权配置管理组件 | 无此步骤 | **设计差异**：当前设计为进件管理直接按产品/进件类型配置调用「风控规则包」。若 v1.1 表示需先查授权配置再决定规则包，则与「授权配置管理」职责（授权文件+规则包绑定）一致；设计上可约定进件阶段规则包 ID 来自产品配置或授权配置表，需在进件/风控配置表中明确数据来源。 |
| **6、客户经理线下办理三合一** | 首步「进件受理线下业务，初始化三合一任务」；所属组件 授信管理组件 | 首步「评级信息填写」等 | v1.1 把「进件受理、初始化三合一任务」作为第一步，与设计「工作流/进件管理按办贷任务类型启动流程、授信管理业务牵头」一致；设计已覆盖。 |
| **7、有权审批人审批** | 阶段单独列出；「业务审批完成」→ 授信管理；「生成合同信息」→ 合同管理；「生成签约任务」→ 签约组件 | 同左 | 与设计一致；工作流驱动审批，审批通过后合同管理+签约生成合同与任务。 |
| **8、客户在线签借款合同** | 「查看需要签订的合同」→ 签约组件；「加载查看合同模板」→ 合同（协议）模版管理组件；「签订借款合同」→ 电子签约平台；更新用信/授信/评级/合同/签约/进件 | 同左 | 设计约定：签约只向合同管理要可展示内容，不直接调模版；v1.1 若表示「签约调模版」需在实现时按设计收敛为「签约→合同管理→（内部）模版」。 |
| **9、客户发起提款** | 签订放款申请书完成调用核心系统放款 → 核心系统；更新进件信息 → 进件管理 | 同左 | 设计已覆盖；放款组件（办贷）调核心，放款成功后通知进件管理。 |
| **10、受托支付** | 仅写「涉及受托支付的需要人工审批」，被调 放款组件 | 同左 | 工作流/审批流 + 放款组件，设计已覆盖。 |

## 三、v1.1 与《产品流程-组件调用场景视图与配置表》的调用链对比

| 阶段 | 场景视图（当前设计） | v1.1 CSV 体现 | 差异与建议 |
|------|----------------------|---------------|------------|
| 1、预授信申请 | 预授信组件 → 风控组件 ← 征信与外部数据管理 | 预授信 → 智能决策（额度测算）；白名单维护被调为空 | 设计链更完整（含征信与外部数据）；v1.1 可补「征信与外部数据管理」为白名单/数据来源。 |
| 3、客户查询产品信息 | 营销方案管理 → 产品/利率/额度/合同管理 | 营销方案 → 产品/利率/额度/合同/产品管理；点击申请 → 贷款申请组件 | 一致；v1.1 明确「点击申请」进入贷款申请组件。 |
| 3、客户发起贷款申请 | 贷款申请 → 客户授权准入 → 授权配置/合同管理（授权展示）/消息/人脸 → 签约（授权）→ 进件管理；可选 用户注册+客户管理 | v1.1 增加：风控（业务准入规则包）在申请内；用户注册为「系统自动注册/认证」 | 见上：提交前风控是否在渠道执行需产品与设计确认；自动注册/认证设计已支持。 |
| 4、系统进件接收 | 进件管理 → 客户管理 → 风控 → 评级 → 风控（额度）→ 授信（初始化）→ 用信（初始化）→ [≤20W]合同管理+签约+进件 / [>20W]进件管理（转线下） | v1.1 增加「查询授权配置管理确认需要执行的风控规则」 | 规则包来源可来自授权配置或产品配置，需在配置表明确。 |
| 8、客户在线签借款合同 | 签约组件 → 用信/授信/评级/合同管理（更新）→ 进件管理 | 签约 → 风控、人脸、查看合同、模版、消息、电子签约 → 用信/授信/评级/合同/签约/进件 | 一致；v1.1 细化了签约环节子步骤（风控、人脸、查看合同、验证、签署、更新各模块）。 |

## 四、设计调整建议汇总

1. **组件命名统一**  
   - 文档与实现中：**智能决策组件** 统一表述为 **风控组件**（及额度测算时 风控+额度管理）。  
   - **电子签约平台** 统一为 **签约组件**。

2. **申请阶段「提交前风控」**（**已纳入设计**）  
   - v1.1 在「客户发起贷款申请」中增加「业务准入规则包：反欺诈、黑名单、产品参数准入」。  
   - **设计已更新**：《产品流程-组件调用场景视图与配置表》中已增加「贷款申请组件 → 风控组件（申请前规则包，按产品配置）」；风控与路由配置表已增加「申请前/进件前」阶段及规则包配置；《组件设计-风控组件与规则触发总览》已增加「申请前/进件前」阶段说明。

3. **进件阶段风控规则包来源**（**已纳入设计**）  
   - **设计已更新**：进件准入规则包 ID 统一由**授权配置（产品+阶段）**提供；进件管理在进件接收时先查授权配置获取规则包 ID 再调用风控。已在场景视图、风控与路由配置表、风控总览、第 4 段进件接收、第 3 段授权配置管理中补充说明；业务阶段示例已含 `INTAKE_ACCESS`（进件准入）、`APPLICATION_PRE_SUBMIT`（申请前）。

4. **签约与合同展示**  
   - 保持现有设计：签约组件不直接调合同（协议）模版管理组件，展示内容由合同管理提供。v1.1 中「加载查看合同模板」在实现时由合同管理统一出口。

5. **预授信阶段**  
   - v1.1 步骤「客户经理维护预产品授信白名」被调为空：设计上主责为预授信组件或批处理；「调用模式测算出预期授信额度」对应风控/预授信规则，设计已覆盖。

|------|------------------|---------------------|------------------|------|
| 1 | 客户经理维护预产品授信白名单 | — | 预授信组件（主责） | 被调可留空或预授信 |
| 1 | 调用模式测算出预期授信额度 | 智能决策组件 | 风控组件（预授信规则/额度测算） | 可与征信与外部数据联动 |
| 2 | 分享二维码 / 扫码 | 营销方案管理组件 | 营销方案管理组件 | 一致 |
| 3 | 产品列表/详情、利率/额度/合同/最高可贷额、点击申请 | 营销方案、利率、额度、合同、产品管理、贷款申请 | 营销方案 + 产品/利率/额度/合同管理、贷款申请 | 一致 |
| 3 | 初始化贷款申请、填写信息 | 贷款申请组件 | 贷款申请组件 | 一致 |
| 3 | 调用风控中心执行规则包（业务准入） | 智能决策组件 | 风控组件 | 见差异2：提交前 vs 进件后 |

| 3 | 提交贷款申请、自动注册/认证、调用进件 | 进件管理、用户注册 | 进件管理、用户注册 | 一致 |
| 4 | 进件接收～场景2 三合一 | 进件、客户、授权配置、智能决策、评级、授信、用信、合同、签约、授信 | 进件管理、客户管理、风控、评级、授信、用信、合同管理、签约、工作流 | 授权配置仅提供规则包 ID 时设计支持 |
| 5～7 | 客户经理客户创建；三合一；审批 | 客户管理；授信、评级、用信、智能决策；授信、合同、签约 | 客户管理；授信、评级、用信、风控、工作流、合同、签约 | 一致 |
| 8 | 发起签约～更新进件状态 | 签约、智能决策、人脸、合同模版、消息、电子签约、用信、授信、评级、合同、签约、进件 | 签约、风控、人脸、合同管理（展示）、消息、签约、用信、授信、评级、合同、进件 | 展示走合同管理 |
| 9～11 | 提款、受托支付、还款 | 放款、安全、智能决策、人脸、消息、核心、进件；放款；还款 | 放款、安全、风控、人脸、消息、核心、进件；工作流+放款；还款 | 一致 |

---

**结论**：v1.1 与现有设计整体一致，主要差异为（1）**申请阶段是否增加提交前风控**、（2）**进件阶段风控规则包是否从授权配置获取**、（3）**组件命名**（智能决策→风控、电子签约→签约）与（4）**签约展示**（统一经合同管理）。按上表调整配置与文档后即可对齐 v1.1。
