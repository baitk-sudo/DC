# 组件设计分析 — 第 5 段：三合一任务与审批（及二合一/单独办贷）

本段覆盖**办贷端主导**的业务阶段：进件转线下后，客户经理补录客户信息、办理办贷任务（**办贷任务类型由配置决定**：三合一（评级+授信+用信）、二合一(授信+用信)、二合一(评级+授信)、单独评级/授信/用信）、有权审批人审批，以及审批通过后生成合同与签约任务。下文以**三合一**为主展开，二合一与单独模式的差异主要为工作流流程模板与节点内调用的组件组合不同。可选地，部分产品在本段前后还有**利率调整、额度调整、额度激活**等客户经理发起并经审批生效的环节。

**本段涉及的组件**（办贷管理端 v2）：

- 客户管理组件（客户经理补录）
- 授信管理组件（三合一业务牵头、授信信息、审批结论、利率/额度调整与激活入口）
- 用信管理组件（用信信息填写与落库）
- 评级管理组件（评级信息填写与落库）
- 额度管理组件（额度测算结果、额度调整、额度激活）
- 风控组件（三合一节点内风控准入、额度测算规则）
- 工作流/审批流组件（三合一任务类型、审批节点、分配与回调）
- 利率与费用组件（利率调整的定价与生效）
- 合同管理组件（审批通过后生成合同草稿/待签）
- 签约组件（生成待签任务）

**已对照产品**：湖北农信-公积金贷款（线上转线下）、湖南农信-公积金贷款、长沙银行-税E贷、长沙银行-政采贷。

**本段涉及的限界上下文**：进件与流程、授信/额度/用信、签约与合同、风控（见《限界上下文与集成关系-ContextMap.md》）。

## 一、客户管理组件（本段：客户经理补录）

### 1.1 本段职责与边界

| 项目 | 说明 |
|------|------|
| **本段职责** | 在办贷端为客户经理提供客户信息补录、修正与保存能力；支持对公/对私、建档与尽调类字段的扩展，供三合一任务与审批依赖的客户主数据使用。 |
| **边界** | 不负责评级、授信、用信、额度的业务逻辑，仅维护客户主数据；补录后的客户信息可被评级、风控、授信等组件在后续节点消费。 |

### 1.2 多产品 / 多模式支持点

- **对公尽调/建档**：政采贷等产品需「客户建档、业务办理、线下尽调」类信息，客户管理通过扩展字段或子集配置支持，避免为每类产品单独建表。
- **与进件关联**：补录可基于进件单关联的客户（临时或已初始化），按进件号或客户号做幂等更新。
- **任务类型适配（三合一/二合一/单独）**：无论办贷任务类型为三合一、二合一（授信+用信 / 评级+授信）还是单独任务，客户信息补录均通过同一补录接口被调用，仅由工作流模板决定在具体流程中是否包含「客户补录」节点及其出现顺序。

### 1.3 关键接口（本段相关）

| 接口/能力 | 说明 | 上游/下游 |
|-----------|------|-----------|
| 客户信息补录/保存 | 客户经理在办贷端填写、修正客户信息并持久化 | 下游：客户表；上游：办贷端前台、进件/工作流触发的补录任务 |

### 1.4 对已分析产品的支撑检查

| 产品 | 本段相关步骤 | 说明 |
|------|--------------|------|
| 湖北/湖南 公积金、税E贷、政采贷 | 客户经理个人客户创建 / 补录客户信息（含尽调） | 客户管理组件提供统一补录与扩展字段能力，满足对公/对私与政采尽调需求 |

## 二、授信管理组件（本段：三合一牵头与审批结论）

### 2.1 本段职责与边界

| 项目 | 说明 |
|------|------|
| **本段职责** | 作为「三合一」业务的业务域牵头方：提供授信信息填写与落库、三合一汇总提交入口（将评级、授信、用信、额度、风控结果一并提交并驱动工作流进入审批节点）；接收审批结论并落库（通过/拒绝/退回）；在审批通过后驱动或协同合同管理、签约组件生成授信合同/借款合同与签约任务。另支持客户经理发起的**利率调整、额度调整、额度激活**的发起与审批结果落库（与利率与费用、额度管理、工作流协同）。 |
| **边界** | 不定义审批流程节点与审批人，由工作流/审批流组件负责；不执行风控规则或额度测算，由风控与额度管理组件执行；合同生成与签约任务由合同管理、签约组件执行，授信管理负责触发或传递业务主键。 |

### 2.2 多产品 / 多模式支持点

- **三合一组合可配置**：不同产品可配置三合一任务内包含哪些子项（必含评级、授信、用信，额度与风控可选），通过工作流任务类型与授信管理提供的「三合一提交」接口协同实现。
- **二合一/单独模式复用同一接口**：当办贷任务类型为二合一(授信+用信或评级+授信)、或单独授信时，工作流模板只拉取对应的子任务节点（如不包含用信/评级）；授信管理的「信息填写/保存」「审批结论写入」「触发生成合同与签约任务」接口保持不变，仅由工作流决定本次任务包含哪些子项。
- **审批结论与进件状态**：审批通过后授信管理写入授信结论，并通知进件管理或工作流更新进件状态（如 APPROVED），以便进件管理触发该状态下的同步策略（生成合同与签约任务可在该策略或工作流节点中执行）。
- **利率调整、额度调整、额度激活**：税E贷/政采贷等产品中，客户经理在办贷端发起上述三类申请，经工作流审批后，授信管理记录审批结果，利率与费用组件、额度管理组件分别执行价格与额度变更；授信管理需提供「发起调整/激活」的入口及与工作流的挂接。

### 2.3 关键接口与上下游关系（本段）

| 接口/能力 | 说明 | 上游/下游 |
|-----------|------|-----------|
| 授信信息填写/保存 | 客户经理录入授信结论、期限、担保方式等并落库 | 下游：授信表；上游：办贷端三合一任务界面 |
| 三合一业务提交 | 汇总当前进件下的评级、授信、用信、额度、风控结果，校验完整性后提交，驱动工作流进入审批节点 | 下游：工作流/审批流组件；上游：办贷端三合一任务界面（经授信管理统一入口） |
| 写入审批结论 | 工作流在审批节点完成时回调，授信管理写入审批通过/拒绝/退回及意见 | 下游：授信表、审批记录；上游：工作流/审批流组件 |
| 审批通过后触发生成合同与签约任务 | 审批通过后，由授信管理触发或由进件管理在 APPROVED 同步策略中调用合同管理、签约组件生成合同与待签任务 | 下游：合同管理组件、签约组件；上游：工作流回调或进件管理 |
| 利率/额度调整与激活发起 | 客户经理发起利率调整、额度调整、额度激活申请，创建调整单并提交工作流审批 | 下游：工作流/审批流组件、利率与费用组件、额度管理组件；上游：办贷端 |

### 2.4 对已分析产品的支撑检查

| 产品 | 本段相关步骤 | 说明 |
|------|--------------|------|
| 湖北/湖南 公积金、税E贷 | 授信信息填写、三合一提交、有权审批人审批完成、生成合同与签约任务 | 授信管理作为三合一业务牵头，与工作流、合同、签约协同，满足 |
| 税E贷、政采贷 | 客户经理发起利率调整、额度调整、额度激活及审批生效 | 授信管理提供发起入口并与工作流、利率与费用、额度管理协同，需在接口与配置上显式支持 |

## 三、用信管理组件（本段）

### 3.1 本段职责与边界

| 项目 | 说明 |
|------|------|
| **本段职责** | 在三合一任务中提供用信信息（用信金额、期限、用途等）的填写与落库；支持客户经理录入或修正，并与授信、额度数据校验一致。 |
| **边界** | 不负责审批流程；不直接生成合同，合同由合同管理组件在审批通过后生成；额度占用在审批通过或签约时由额度管理组件执行，用信管理记录用信申请/用信结论。 |

**任务类型适配（三合一/二合一/单独）**：

- 在**三合一任务（评级+授信+用信）**下，用信管理在同一任务中参与填写与落库；
- 在**二合一（授信+用信）**任务下，用信管理与授信管理共同作为本次办贷任务的子节点，被工作流模板拉起；
- 在**二合一（评级+授信）**任务下，本段不参与该任务，本次只处理评级+授信；用信由后续单独的「额度项下贷款申请/用信任务」复用本组件接口完成；
- 在**单独用信任务**下（如额度项下贷款申请、用信调整等），工作流模板仅拉起用信管理相关节点，本段接口仍保持不变。

### 3.2 关键接口（本段相关）

| 接口/能力 | 说明 | 上游/下游 |
|-----------|------|-----------|
| 用信信息填写/保存 | 客户经理录入或修正用信金额、期限、用途等并落库 | 下游：用信表；上游：办贷端三合一任务界面 |
| 按进件/授信校验与落库 | 用信数据与当前授信、额度校验，支持按进件号幂等更新 | 下游：用信表；上游：三合一提交、进件同步策略 |

### 3.3 对已分析产品的支撑检查

| 产品 | 本段相关步骤 | 说明 |
|------|--------------|------|
| 湖北/湖南 公积金、税E贷、政采贷 | 用信信息填写、三合一业务提交中含用信 | 用信管理提供填写与落库接口，满足 |

## 四、评级管理组件（本段）

### 4.1 本段职责与边界

| 项目 | 说明 |
|------|------|
| **本段职责** | 在三合一任务中提供评级信息的填写、调整与落库；支持客户经理手工录入或触发模型重算，结果供授信、风控、额度测算使用。 |
| **边界** | 不执行审批；不直接写授信/额度，评级结果由授信管理、额度管理在业务逻辑中消费。 |

**任务类型适配（三合一/二合一/单独）**：

- 在**三合一任务**下，评级管理作为任务内子节点参与填写/重算并落库；
- 在**二合一（评级+授信）**任务下，仅包含评级与授信两个子节点，本段接口同样被调用；
- 在**二合一（授信+用信）**或**单独用信任务**下，本段不参与该任务，评级可在前一阶段或单独评级任务中完成，仍复用本段接口。

### 4.2 关键接口（本段相关）

| 接口/能力 | 说明 | 上游/下游 |
|-----------|------|-----------|
| 评级信息填写/保存 | 客户经理录入或调整评级结果及理由，或触发模型重算后落库 | 下游：评级表；上游：办贷端三合一任务界面 |
| 按进件幂等更新 | 支持按进件号对评级结果做幂等写入，与早同步/晚同步策略兼容 | 下游：评级表；上游：三合一提交、进件同步策略 |

### 4.3 对已分析产品的支撑检查

| 产品 | 本段相关步骤 | 说明 |
|------|--------------|------|
| 湖北/湖南 公积金、税E贷、政采贷 | 评级信息填写、三合一业务提交中含评级 | 评级管理提供填写与落库能力，满足 |

## 五、额度管理组件（本段）

### 5.1 本段职责与边界

| 项目 | 说明 |
|------|------|
| **本段职责** | 在三合一任务中接收额度测算结果并落库（预占或待审批占用）；支持客户经理发起的**额度调整**与**额度激活**申请，经工作流审批通过后更新额度并留痕；与授信、用信数据联动，避免超授信或超可用额。 |
| **边界** | 不定义审批流程，审批由工作流承载；额度测算规则可由风控组件执行，额度管理负责结果落库与额度台账维护。 |

### 5.2 多产品 / 多模式支持点

- **额度调整**：支持「额度调整申请」与工作流审批挂接，审批通过后更新授信项下额度并留痕，供多产品复用。
- **额度激活**：支持额度状态（如预占、未激活、激活、冻结）及「额度激活」流程（可配置是否需审批），满足税E贷/政采贷等产品在授信通过后需单独激活额度的场景。
- **适配二合一/单独任务**：在二合一或单独授信/用信任务中，额度管理组件同样通过「额度测算结果落库」「额度调整/激活」接口被调用，仅由工作流模板控制本次任务是否涉及额度调整或激活步骤。

### 5.3 关键接口（本段相关）

| 接口/能力 | 说明 | 上游/下游 |
|-----------|------|-----------|
| 额度测算结果落库 | 接收风控或三合一节点传入的额度测算结果，写入额度台账（预占或待生效） | 下游：额度表；上游：风控组件、三合一任务 |
| 额度调整申请与生效 | 客户经理发起额度调整，工作流审批通过后额度管理更新额度并留痕 | 下游：额度表；上游：授信管理（发起）、工作流（审批回调） |
| 额度激活申请与生效 | 客户经理发起额度激活，按配置经工作流审批或直接生效，更新额度状态 | 下游：额度表；上游：授信管理（发起）、工作流（可选审批） |

### 5.4 对已分析产品的支撑检查

| 产品 | 本段相关步骤 | 说明 |
|------|--------------|------|
| 湖北/湖南 公积金、税E贷、政采贷 | 测算额度结果、三合一内含额度；税E贷/政采贷 额度调整、额度激活 | 额度管理支持测算结果落库及调整/激活流程，满足；需显式支持审批式调整与激活状态 |

## 六、风控组件（本段：三合一节点内）

### 6.1 本段职责与边界

| 项目 | 说明 |
|------|------|
| **本段职责** | 在三合一任务节点内执行客户风控准入规则（及可选额度测算规则），返回通过/拒绝/转人工等结果，供三合一提交前的校验或落库使用。 |
| **边界** | 不维护授信/用信/评级/额度业务表，仅输出规则结果；外部数据通过征信与外部数据管理组件获取。 |

**任务类型适配（三合一/二合一/单独）**：风控组件在所有包含「办贷风控准入」或「额度测算」子步骤的办贷任务模板下被调用——三合一任务内通常统一执行一次；二合一任务可按配置选择在授信+用信或评级+授信节点前后执行；单独授信/用信任务可仅执行与本次任务相关的规则包，风控接口本身不区分任务类型，由工作流模板与规则包绑定关系控制是否、何时调用。

### 6.2 关键接口（本段相关）

| 接口/能力 | 说明 | 上游/下游 |
|-----------|------|-----------|
| 三合一节点内风控准入 | 按进件/客户/产品执行风控规则包，返回准入结果 | 下游：征信与外部数据管理（取数）；上游：三合一任务节点（授信管理或工作流调用） |
| 额度测算规则执行（可选） | 部分产品在三合一中由风控执行额度测算规则，结果交额度管理落库 | 下游：额度管理组件；上游：三合一任务 |

### 6.3 对已分析产品的支撑检查

| 产品 | 本段相关步骤 | 说明 |
|------|--------------|------|
| 湖北/湖南 公积金、税E贷、政采贷 | 客户风控准入、测算额度结果（规则侧） | 风控组件提供规则执行能力，满足 |

## 七、工作流/审批流组件（本段）

### 7.1 本段职责与边界

| 项目 | 说明 |
|------|------|
| **本段职责** | 定义并执行**多种办贷流程模板**（三合一、二合一(授信+用信)、二合一(评级+授信)、单独评级/授信/用信），按进件管理传入的办贷任务类型或流程模板 ID 启动对应实例；节点内串联客户经理办理评级、授信、用信、额度、风控等（按模板组合）；定义审批节点（有权审批人审查审批）；在审批节点完成时回调进件管理或授信管理更新状态，并触发生成合同与签约任务；支持利率调整、额度调整、额度激活的审批流挂接。 |
| **边界** | 不执行业务规则，不直接写入客户/授信/用信/评级/额度业务表；业务写入由各业务组件在节点内或回调时完成；工作流只负责任务定义、按办贷任务类型启动对应模板、流转、分配与回调。 |

### 7.2 多产品 / 多模式支持点

- **办贷任务类型与流程模板**：支持三合一（评级+授信+用信+额度+风控）、二合一(授信+用信)、二合一(评级+授信)、单独模式等多种办贷流程模板，由进件管理在转线下时根据配置传入；节点内按模板依次调用各业务组件接口；不同产品可配置不同的办贷任务类型及子步骤或校验顺序。
- **审批流配置**：支持多级审批、会签、转签、退回、超时等；审批通过/拒绝时回调授信管理、进件管理，驱动后续生成合同与签约任务或进件状态更新。
- **调整类审批流**：利率调整、额度调整、额度激活可配置为独立审批流类型，与授信管理、利率与费用、额度管理的发起与生效接口对接。

### 7.3 关键接口与上下游关系（本段）

| 接口/能力 | 说明 | 上游/下游 |
|-----------|------|-----------|
| 三合一任务执行 | 客户经理在任务内办理评级、授信、用信、额度、风控，工作流驱动节点内调用各组件并收集结果 | 下游：评级、授信、用信、额度、风控组件；上游：进件管理（任务下发） |
| 审批节点执行 | 有权审批人审批，填写意见并通过/拒绝/退回，工作流流转并回调 | 下游：授信管理（写入审批结论）、进件管理（更新状态）；上游：办贷端审批界面 |
| 审批通过后触发 | 审批通过时触发生成合同与签约任务（或由进件管理在 APPROVED 同步策略中执行） | 下游：合同管理组件、签约组件；上游：工作流引擎 |
| 调整/激活审批流 | 利率调整、额度调整、额度激活的审批流定义与执行，审批通过后回调业务组件生效 | 下游：授信管理、利率与费用、额度管理；上游：办贷端发起 |

### 7.4 对已分析产品的支撑检查

| 产品 | 本段相关步骤 | 说明 |
|------|--------------|------|
| 湖北/湖南 公积金、税E贷、政采贷 | 三合一任务、有权审批人审批、生成合同与签约任务；税E贷/政采贷 利率/额度调整与激活审批 | 工作流支持三合一组合任务与审批节点及调整类审批流，满足 |

## 九、合同管理组件与签约组件（本段：审批通过后）

### 9.1 本段职责与边界

| 组件 | 本段职责 | 边界 |
|------|----------|------|
| **合同管理组件** | 在审批通过后（或进件状态 APPROVED 的同步策略中）按产品/授信生成合同实例（授信合同或借款合同，草稿/待签状态），并关联进件、客户、授信、用信；生成实例时消费合同模版管理组件获取模板 | 不执行审批；签约展示内容由合同管理统一提供，签约组件不直接调模版管理 |
| **签约组件** | 根据生成的待签合同，创建签约任务并关联合同、客户、进件，供**面客渠道**或**办贷端 PAD** 现场签约；签约组件为公共能力，两种入口共用同一套任务与状态 | 不执行业务审批；任务生成后客户签署流程在第 6 段（签约与合同管理）详述 |

### 9.2 关键接口（本段相关）

| 接口/能力 | 说明 | 上游/下游 |
|-----------|------|-----------|
| 生成合同（审批通过后） | 按产品、授信、用信信息生成合同实例，状态为待签 | 下游：合同表；上游：授信管理（触发）、进件管理（同步策略）、工作流（回调） |
| 生成签约任务 | 创建待签任务，关联合同、进件、客户，供**面客渠道**或**办贷 PAD** 拉取并执行签署（签约组件为公共能力，统一支撑两种签约入口） | 下游：签约任务表；上游：合同管理（合同就绪后）、签约组件、工作流或进件管理 |

### 9.3 对已分析产品的支撑检查

| 产品 | 本段相关步骤 | 说明 |
|------|--------------|------|
| 湖北/湖南 公积金、税E贷、政采贷 | 审批完成后生成合同信息、生成签约任务 | 合同管理、签约组件在审批通过后生成合同与任务，满足；合同类型（授信合同/借款合同）在第 6 段细化 |

## 十、本段组件协作关系小结

```text
[进件转人工]
    -> 工作流：根据配置生成办贷任务（办贷任务类型：三合一 / 二合一 / 单独）
        -> 客户经理办理（子节点随办贷任务类型不同）：
           - 三合一（评级+授信+用信）：
                 客户管理（补录）
              -> 评级管理（填写）
              -> 授信管理（填写）
              -> 用信管理（填写）
              -> 风控组件（准入） / 额度管理（测算结果落库）
           - 二合一（授信+用信）：
                 客户管理（补录，可选）
              -> 授信管理（填写）
              -> 用信管理（填写）
              -> 风控组件 / 额度管理（按产品配置）
           - 二合一（评级+授信）：
                 客户管理（补录，可选）
              -> 评级管理（填写）
              -> 授信管理（填写）
              -> 风控组件 / 额度管理（按产品配置；用信在后续单独任务中处理）
           - 单独任务（仅评级 / 授信 / 用信等）：
                 客户管理（补录，可选）
              -> 对应单一组件（评级 / 授信 / 用信），复用本段相同接口
        -> 授信管理（提交本次办贷任务：三合一 / 二合一 / 单独）
        -> 工作流：进入审批节点
        -> 有权审批人审批
        -> 工作流：审批完成回调 --> 授信管理（写入审批结论） / 进件管理（更新进件状态，如 APPROVED）
        -> 进件管理（状态 APPROVED）--> 同步策略 / 工作流触发
        -> 合同管理（生成合同）--> 签约组件（生成签约任务）--> 客户待签（第 6 段）
```

**可选分支（税E贷/政采贷）**：客户经理发起利率调整、额度调整、额度激活 → 工作流审批 → 利率与费用组件、额度管理组件生效。

## 十一、办贷任务类型对照表（三合一 / 二合一 / 单独）

| 办贷任务类型 | 流程模板特征 | 本段组件参与情况（示意） |
|--------------|--------------|---------------------------|
| 三合一（评级+授信+用信） | 单个任务内同时承载评级、授信、用信（+额度/风控），由授信管理统一提交并进入审批节点 | 客户管理补录 + 评级管理 + 授信管理 + 用信管理 + 额度管理 + 风控组件；审批通过后触发合同管理、签约组件 |
| 二合一（授信+用信） | 同一任务内包含授信与用信，用信与授信审批合并处理 | 客户管理补录 + 授信管理 + 用信管理 + 额度管理/风控（按配置）；评级可在前一阶段或单独任务中完成 |
| 二合一（评级+授信） | 同一任务内包含评级与授信；用信在后续「额度项下贷款申请/用信任务」中单独处理 | 客户管理补录 + 评级管理 + 授信管理 + 额度管理/风控（按配置）；后续单独用信任务复用用信管理接口 |
| 单独任务（仅评级/仅授信/仅用信等） | 工作流模板只拉取一个子任务节点（如单独授信补批、单独用信调整） | 对应的单个组件（评级/授信/用信）通过本段定义的接口被调用，审批流与进件状态更新逻辑与三合一一致，由工作流模板与进件管理配置决定 |

## 十一、与前序及后续段落衔接

- **与第 4 段（进件与风控准入）**：办贷任务（三合一/二合一/单独）由进件转线下后进件管理根据配置解析办贷任务类型并协同工作流启动对应流程模板；进件状态在审批通过后由工作流或授信管理回调进件管理更新为 APPROVED，并触发同步策略。
- **与第 6 段（签约与合同管理）**：本段结束于「生成合同与签约任务」，客户在线签署、合同生效及与进件同步策略（SIGNED）的衔接在第 6 段展开。
- **与《进件中心-业务数据同步策略设计》**：审批通过后若配置为在 APPROVED 状态生成合同与签约任务，可由进件管理在该状态迁移时通过同步策略调用合同管理、签约组件完成，与本段设计一致。

本文件为「按业务办理过程顺序」组件分析的第 5 段；第 6 段为**签约与合同管理**（客户在线签约、合同生效、多合同类型与多轮签约、与同步策略衔接）。
