# 组件设计分析 — 第 8 段：还款

本段覆盖**放款完成后**的还款全流程：还款计划生成与查询、客户在渠道端发起还款（正常还款、提前结清、部分提前还款）、还款指令提交与办贷端处理、与核心/账务系统对接完成入账与利息/罚息结算、用信与还款计划状态更新，以及**逾期**状态维护、还款提醒与逾期提示等贷后还款相关能力。

**本段涉及的组件**（办贷管理端 v2 + 渠道端）：

- 还款组件（渠道端：还款计划展示、主动还款与提前结清；办贷端：还款指令处理、核心/账务对接、还款计划与用信状态更新）
- 用信管理组件（用信/贷款明细、放款信息，供还款计划与还款记录关联；结清后状态更新）
- 利率与费用组件（计息、罚息、本金分解、还款计划测算参数）
- 批处理组件（还款计划批量生成、逾期状态批量更新、到期日/扣款日批处理）
- 产品管理组件（还款方式、计息方式等产品参数）
- 消息中心组件（还款提醒、逾期通知）
- 贷后管理/预警组件（逾期与还款行为作为贷后监控输入，本段仅数据消费关系）

**已对照产品**：湖北农信-公积金贷款（线上/线上转线下）、湖南农信-公积金贷款、长沙银行-税E贷、长沙银行-政采贷。

**本段涉及的限界上下文**：支付与账务集成、授信/额度/用信、批处理与调度、产品与营销、通用安全与消息（见《限界上下文与集成关系-ContextMap.md》）。

## 一、还款组件（渠道端）

### 1.1 职责与边界

| 项目 | 说明 |
|------|------|
| **职责** | 向客户展示贷款账户列表、还款计划（应还日期、应还本金/利息/费用、已还/未还）、历史还款记录；支持客户发起**正常还款**、**提前结清**、**部分提前还款**等操作，采集还款金额与还款方式后生成还款指令并提交办贷端；展示还款结果与剩余应还信息；支持还款提醒、逾期提示等客户服务能力（可调用消息中心或由办贷端统一推送）。 |
| **边界** | 不执行计息、罚息或账务处理，计息与还款计划由利率与费用组件、还款组件（办贷端）或批处理提供；不直接调用核心/账务系统，还款指令由办贷端还款组件接收并落地；逾期状态与罚息由办贷端与批处理维护，渠道端仅展示。 |

### 1.2 多产品 / 多模式支持点

- **多种还款方式**：等额本息、等额本金、按期付息到期还本、随借随还等，由产品管理配置，还款计划与应还金额由办贷端根据产品参数与利率与费用组件计算，渠道端按约定展示。
- **正常还款 vs 提前结清 vs 部分提前还款**：渠道端支持三种入口，提交时携带还款类型与金额（或“全部结清”标识），办贷端按类型执行不同逻辑（正常扣款、提前结清重算、部分提前还款后重算剩余计划）。
- **还款提醒与逾期提示**：可按产品/客户配置还款日前提醒、逾期后提醒，由消息中心发送短信/站内信等；渠道端可展示“即将到期”“已逾期”等状态，数据来源于办贷端还款计划与逾期状态。

### 1.3 关键接口与上下游关系

| 接口/能力 | 说明 | 上游/下游 |
|-----------|------|-----------|
| 贷款账户与还款计划查询 | 按客户查询其名下贷款/用信列表及每笔的还款计划（应还日、应还本金/利息/费用、已还、逾期天数等） | 下游：还款组件（办贷端）或统一还款计划服务；上游：渠道 |
| 历史还款记录查询 | 按客户/用信查询已发生的还款流水、冲账明细 | 下游：还款组件（办贷端）；上游：渠道 |
| 发起还款（正常/提前结清/部分提前） | 客户选择还款类型、金额（或全部）、扣款账户等，提交还款指令至办贷端 | 下游：还款组件（办贷端）；上游：渠道 |
| 还款结果查询 | 查询某笔还款指令的状态（处理中/成功/失败）及失败原因 | 下游：还款组件（办贷端）；上游：渠道 |
| 还款提醒/逾期提示展示 | 展示即将到期或已逾期期次，数据来源于办贷端；推送由消息中心执行 | 下游：还款组件（办贷端）、消息中心；上游：渠道 |

### 1.4 对已分析产品的支撑检查

| 产品 | 本段相关步骤 | 说明 |
|------|--------------|------|
| 湖北/湖南 公积金、税E贷、政采贷 | 客户发起还款申请 | 渠道端还款组件提供还款计划查看与主动还款入口，办贷端负责计息、入账与状态更新 |

## 二、还款组件（办贷端）

### 2.1 职责与边界

| 项目 | 说明 |
|------|------|
| **职责** | 接收渠道或内部发起的还款指令（正常还款、提前结清、部分提前还款）；根据用信与还款计划、利率与费用组件提供的计息/罚息规则，计算应收本金、利息、费用、罚息（若有逾期）；与核心/账务系统对接，完成扣款、入账、冲账；更新还款计划执行情况（已还期次、剩余本金等）与用信管理组件中的用信状态（如结清）；支持减免、冲正等运营处理（若产品需要）。 |
| **边界** | 不维护客户/授信/用信主数据，用信与贷款明细由用信管理组件提供；计息与罚息规则、还款方式由利率与费用组件与产品管理提供，还款组件负责按规则计算并调用核心；还款计划可由本组件或批处理组件在放款后生成，需与用信、产品、利率参数一致。 |

### 2.2 多产品 / 多模式支持点

- **多还款方式**：等额本息、等额本金、按期付息到期还本、随借随还等，根据产品管理配置的还款方式与利率与费用组件的计息方式，生成还款计划并支持后续变更（如部分提前还款后重算）。
- **逾期与罚息**：支持按产品配置逾期罚息规则（利率、起算日、计息基数），与利率与费用组件协同；逾期状态可由批处理日终更新或实时更新，还款组件在客户还款时按“先息后本”“先逾期后正常”等冲账顺序计算应还金额。
- **提前结清与部分提前还款**：提前结清时结清全部剩余本金与应付利息/费用；部分提前还款时按约定重算剩余期数与每期应还额，并更新还款计划。
- **与核心/账务对接**：还款组件负责组包还款交易、调用核心扣款与入账，处理成功/失败回盘；支持重试与对账，确保本系统还款计划执行情况与账务一致。

### 2.3 关键接口与上下游关系

| 接口/能力 | 说明 | 上游/下游 |
|-----------|------|-----------|
| 还款计划查询 | 按客户/用信返回还款计划明细（应还日、应还本金/利息/费用、已还、逾期等），供渠道与内部查询 | 下游：还款计划表、用信管理（用信明细）；上游：还款组件（渠道）、批处理、贷后等 |
| 接收还款指令 | 接收渠道或内部提交的还款请求（正常/提前结清/部分提前），落库为还款指令并进入处理流程 | 下游：还款指令表；上游：还款组件（渠道）、批处理（代扣） |
| 应还金额计算 | 根据还款类型、当前还款计划、利率与费用规则计算本次应还本金、利息、费用、罚息 | 下游：利率与费用组件；上游：还款组件（办贷） |
| 调用核心还款 | 组装还款交易报文，调用核心/账务系统执行扣款与入账，处理回盘 | 下游：核心系统/账务系统；上游：还款组件（办贷） |
| 还款计划与用信状态更新 | 还款成功后更新还款计划（已还期次、剩余本金）、用信状态（正常/逾期/结清等） | 下游：还款计划表、用信管理组件；上游：还款组件（办贷） |
| 减免/冲正（可选） | 支持运营发起的利息减免、还款冲正等，按产品与权限配置 | 下游：还款计划表、核心/账务；上游：办贷端运营、工作流（若需审批） |

### 2.4 对已分析产品的支撑检查

| 产品 | 本段相关步骤 | 说明 |
|------|--------------|------|
| 湖北/湖南 公积金、税E贷、政采贷 | 客户发起还款申请；入账、冲账、减免等 | 办贷端还款组件负责计息、罚息、核心对接与还款计划/用信状态更新，满足多产品还款规则 |

## 三、用信管理组件（本段：还款数据基础与结清状态）

### 3.1 本段职责与边界

| 项目 | 说明 |
|------|------|
| **本段职责** | 为还款组件提供用信/贷款明细（放款金额、起息日、到期日、利率、还款方式等），供还款计划生成与还款指令校验；在还款组件完成还款或提前结清后，接收状态回写，更新用信记录的还款状态（如正常、逾期、结清）；为贷后管理、监管报送提供准确的用信与还款状态。 |
| **边界** | 不生成还款计划、不执行计息或扣款；还款计划与执行明细由还款组件维护，用信管理仅维护用信维度的汇总状态（如结清、未结清、当前逾期期数等，可按实现约定由还款组件回写）。 |

### 3.2 关键接口（本段相关）

| 接口/能力 | 说明 | 上游/下游 |
|-----------|------|-----------|
| 用信明细查询 | 按客户/授信/用信返回贷款明细（金额、期限、利率、还款方式、放款日等），供还款计划生成与展示 | 下游：无；上游：还款组件、批处理 |
| 用信状态更新（还款回写） | 还款或结清后更新用信记录的还款状态、结清日期等 | 下游：用信表；上游：还款组件（办贷） |

## 四、利率与费用组件（本段：计息、罚息与还款计划参数）

### 4.1 本段职责与边界

| 项目 | 说明 |
|------|------|
| **本段职责** | 为还款计划生成提供计息方式、利率、费用项目与计费规则；为还款组件在客户还款时提供**应还利息、罚息、费用**的计算依据（如利率、罚息率、计息基数、费用摊销规则）；支持按产品、客户、用信维度的差异化定价在还款环节的落地。 |
| **边界** | 不生成还款计划表、不执行扣款；仅提供“按某用信、某日期、某还款类型”计算出的利息/费用/罚息结果，还款组件负责调用并写入还款计划与还款流水。 |

### 4.2 关键接口（本段相关）

| 接口/能力 | 说明 | 上游/下游 |
|-----------|------|-----------|
| 还款计划测算参数 | 按产品/用信提供计息方式、利率、费用项、罚息规则等，供还款计划生成（批处理或还款组件）使用 | 下游：无；上游：批处理、还款组件（办贷） |
| 应还利息/费用/罚息计算 | 按用信、还款日、还款类型（正常/提前结清/部分提前）、当前本金与逾期天数等，计算本次应还利息、费用、罚息 | 下游：无；上游：还款组件（办贷） |

## 五、批处理组件（本段：还款计划生成与逾期状态更新）

### 5.1 本段职责与边界

| 项目 | 说明 |
|------|------|
| **本段职责** | 在放款完成后（或按日/按批）根据用信明细、产品还款方式与利率与费用参数，**批量生成或更新还款计划**；在约定时点（如日终）**批量更新逾期状态**（将到期未还的期次标记为逾期、触发罚息计算）；可选支持**批量代扣**（如约定还款日自动扣款），将扣款指令提交还款组件处理。 |
| **边界** | 不直接调用核心扣款时，由还款组件统一对接核心；还款计划的数据结构、存储位置可与还款组件约定（存在还款组件或独立计划表），批处理只负责按规则生成/更新计划与逾期标记。 |

### 5.2 关键接口（本段相关）

| 接口/能力 | 说明 | 上游/下游 |
|-----------|------|-----------|
| 还款计划批量生成 | 根据新增用信/放款记录、产品与利率参数，生成还款计划明细 | 下游：还款计划表、用信管理（若需回写计划概要）；上游：用信管理、产品管理、利率与费用 |
| 逾期状态批量更新 | 日终或定时将“应还日已过且未还”的期次更新为逾期，并触发罚息计算或标记 | 下游：还款计划表、利率与费用（罚息）；上游：批处理调度 |
| 批量代扣（可选） | 在约定还款日对指定客户/用信发起扣款指令，提交还款组件执行 | 下游：还款组件（办贷）；上游：批处理调度 |

### 7.1 本段职责与边界

| 项目 | 说明 |
|------|------|

## 八、贷后管理/预警组件（本段：数据消费）

### 8.1 本段职责与边界

| 项目 | 说明 |
|------|------|
| **本段职责** | 贷后管理/预警组件消费**还款计划执行情况**与**逾期状态**，用于贷后监控、风险预警与催收策略；不在本段执行主动操作，仅读取还款组件、用信管理提供的数据。 |
| **边界** | 还款数据的生产与维护由还款组件、用信管理、批处理负责；贷后段落将单独展开贷后管理/预警组件的完整设计。 |

## 十、多产品还款模式小结

| 模式 | 说明 | 典型产品 |
|------|------|----------|
| 等额本息/等额本金 | 按期还本付息，还款计划固定或随部分提前还款重算 | 公积金、税E贷、政采贷等 |
| 按期付息到期还本 | 每期还息，到期还本 | 部分经营贷、流动资金贷 |
| 随借随还 | 按实际占用天数计息，还款时本息一并结清或按约定还 | 循环贷、随借随还类产品 |
| 提前结清/部分提前还款 | 支持提前结清全部或部分本金，重算剩余计划或结清 | 多产品共用，规则可配置 |
| 逾期与罚息 | 到期未还标记逾期，按产品规则计罚息 | 多产品共用，利率与费用组件提供规则 |

还款组件（渠道 + 办贷）、利率与费用、批处理、用信管理通过**产品配置**（还款方式、计息方式、罚息规则、费用项）与**统一接口**支持上述多产品场景，无需为每种产品写死还款逻辑。

## 十一、与前序及后续段落衔接

- **与第 7 段（放款与受托支付审批）**：放款完成后，用信管理落好用信与放款信息，批处理或还款组件据此生成还款计划；本段从“客户查询还款计划、发起还款”开始。
- **与利率与费用组件**：放款时已确定利率与费用，还款阶段按同一套参数计息与收费；若存在利率调整，需在授信/用信维度记录调整历史，还款计算时按约定取适用利率。
- **与后续贷后管理/预警段落**：还款与逾期数据为贷后监控与预警的输入；本段仅保证还款计划、还款执行、逾期状态数据准确可查，贷后段落将展开催收、预警规则与工单。

本文件为「按业务办理过程顺序」组件分析的第 8 段；后续段落可继续分析**贷后管理/预警**、**监管报送与报表**、**批处理在流程中的衔接**等。
